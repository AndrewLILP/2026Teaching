Part 1 · Concept (10 min)

How a Game Thinks

Every video game — no matter how complex — runs the same fundamental structure: a game loop. The loop keeps running until a win or loss condition is met. In a board game, the loop maps very cleanly to turns.

Study the diagram below, then read the pseudocode that expresses the same logic.

END

Part 2 · NSW Pseudocode (15 min)

Writing the Game Loop in Pseudocode

Before writing C#, we express the algorithm in NSW pseudocode — plain structured English that any programmer can read, regardless of language. Pseudocode focuses on what happens, not how the language syntax works.

NSW Pseudocode conventions used here: Keywords in UPPERCASE (BEGIN, END, WHILE, IF, THEN, ELSE, FOR EACH). Assignment uses ←. Output uses OUTPUT. Conditions use plain English inside the keyword structure.
NSW Pseudocode
BEGIN GameLoop
  // Run until the game ends
  WHILE winner = FALSE

    // Step 1: Roll the dice
    roll ← RollDice()

    // Step 2: Move the current player
    currentPlayer.position ←
      (currentPlayer.position + roll)
      MOD boardLength

    // Step 3: Resolve the space
    space ← board[currentPlayer.position]
    IF space.type = "Property" THEN
      IF space.owner = NULL THEN
        OfferPurchase(currentPlayer, space)
      ELSE IF space.owner ≠ currentPlayer THEN
        ChargeRent(currentPlayer, space)
      END IF
    ELSE IF space.type = "Tax" THEN
      currentPlayer.money ←
        currentPlayer.money - space.taxAmount
    ELSE IF space.type = "Go" THEN
      currentPlayer.money ←
        currentPlayer.money + goSalary
    END IF

    // Step 4: Check win condition
    IF currentPlayer.money < 0 THEN
      EliminatePlayer(currentPlayer)
    END IF
    IF activePlayers.count = 1 THEN
      winner ← TRUE
      DisplayWinner(activePlayers[0])
    END IF

    // Step 5: Advance to next player
    currentPlayerIndex ←
      (currentPlayerIndex + 1)
      MOD activePlayers.count

  END WHILE
END GameLoop
C# (Unity 6)
// GameManager.cs — game loop entry
void StartTurn()
{
    // Step 1: Roll the dice
    int roll = RollDice();

    // Step 2: Move the current player
    Player p = activePlayers[currentIndex];
    p.position = (p.position + roll)
                 % boardSpaces.Length;

    // Step 3: Resolve the space
    BoardSpace space = boardSpaces[p.position];
    if (space.spaceType ==
        SpaceType.Property)
    {
        if (space.owner == null)
            OfferPurchase(p, space);
        else if (space.owner != p)
            ChargeRent(p, space);
    }
    else if (space.spaceType ==
             SpaceType.Tax)
    {
        p.money -= space.taxAmount;
    }
    else if (space.spaceType ==
             SpaceType.Go)
    {
        p.money += goSalary;
    }

    // Step 4: Check win condition
    if (p.money < 0)
        EliminatePlayer(p);

    if (activePlayers.Count == 1)
    {
        DisplayWinner(activePlayers[0]);
        return;
    }

    // Step 5: Next player
    currentIndex =
        (currentIndex + 1)
        % activePlayers.Count;
}

Notice how each line of pseudocode maps directly to one or two lines of C#. The logic is identical — only the syntax changes.
