Part 1 · The Dice Algorithm (10 min)

Two Dice, One Algorithm

Rolling two dice in code is simpler than it sounds — two calls to Random.Range(1, 7) added together. But why 7 and not 6? And why does rolling two dice matter instead of one die giving 2–12 directly?

NSW Pseudocode
BEGIN RollDice
  die1 ← RANDOM(1, 6)
  die2 ← RANDOM(1, 6)
  total ← die1 + die2
  OUTPUT "Rolled: " + die1
       + " + " + die2
       + " = " + total
  RETURN total
END RollDice
C# — GameManager.cs
public int RollDice()
{
  int d1 = Random.Range(1, 7);
  int d2 = Random.Range(1, 7);
  int total = d1 + d2;
  Debug.Log($"Rolled: {d1} + {d2}
            = {total}");
  return total;
}
Why Random.Range(1, 7) and not (1, 6)? Unity's Random.Range for integers is inclusive of the minimum and exclusive of the maximum. So Range(1, 7) gives 1, 2, 3, 4, 5, or 6 — exactly what a physical die produces. Range(1, 6) would give 1–5 only, which is wrong.

Why two dice instead of one 2–12 roll?

A single Random.Range(2, 13) gives each value from 2–12 an equal probability. Two dice do not — rolling a 7 is six times more likely than rolling a 2. This unequal distribution is intentional in Monopoly; it affects which spaces get landed on most. Try the simulator below and watch what happens over many rolls.

Part 2 · Board Wrap with Modulo (10 min)

The Looping Board Problem

After the last space, the next step should land on space 0 (Go) — not cause an IndexOutOfRangeException. The solution is one of the most elegant tools in programming: the modulo operator (%).

Modulo gives you the remainder after division. When a player's position exceeds the board length, modulo automatically wraps it back to the correct index.

NSW Pseudocode
// Board has 24 spaces (indices 0–23)
// Player is on space 21, rolls 6

rawPosition ← 21 + 6  // = 27
newPosition ← 27 MOD 24 // = 3

// Player lands on space 3 ✓
// They crossed Go: collect salary

IF newPosition < oldPosition THEN
  player.money ← player.money + goSalary
END IF
C# — GameManager.cs
// boardSpaces.Length = 24
// Player on space 21, rolls 6

int raw = p.position + roll; // 27
int newPos = raw
    % boardSpaces.Length;   // 3

// Detect Go crossing
if (newPos <= p.position
    && raw != p.position)
{
    p.AddMoney(goSalary);
    ShowMessage(p.playerName
      + " passed Go! +$"
      + goSalary);
}

Part 3 · Writing the Code (25 min)

MovePlayer() and StartTurn()

Open GameManager.cs in your code editor. The methods below replace the // TODO (Lesson 5) comment stubs. Copy each method carefully — the order of operations matters.

⚠ Before you copy: Read the full method first, then copy it. Pasting code you haven't read is the leading cause of bugs that take 20 minutes to diagnose. Spend 2 minutes reading, save 20 minutes debugging.
Method 1 of 2 — Entry point for each player's turn GameManager.cs 
// Called by the Roll Dice button in the UI.
// Guards against rolling again mid-turn.
public void StartTurn()
{
    // Ignore button presses while we're waiting for player input (e.g. buy/skip)
    if (waitingForInput || gameOver) return;

    Player current = activePlayers[currentIndex];
    int    roll    = RollDice();

    ShowMessage($"{current.playerName} rolled {roll}!");
    MovePlayer(current, roll);
}
Method 2 of 2 — Move token and detect Go crossing GameManager.cs 
public void MovePlayer(Player p, int roll)
{
    int oldPosition = p.position;

    // Calculate new position; % wraps the index around the board
    int newPosition = (p.position + roll) % boardSpaces.Length;

    // Detect Go crossing: new index is LESS than old (wrapped) or exactly 0
    // Also guards against a roll of 0 (shouldn't happen, but defensive)
    bool passedGo = (newPosition <= oldPosition) && (roll > 0);

    if (passedGo)
    {
        p.AddMoney(goSalary);
        ShowMessage($"{p.playerName} passed Go! Collected ${goSalary}.");
        Debug.Log($"{p.playerName} passed Go. Money now: ${p.money}");
    }

    // Update the data
    p.position = newPosition;

    // Move the 3D token to the new space's world position
    // We offset Y by +0.6 so the token sits on top of the flat board tile
    if (p.token != null && boardSpaces[newPosition] != null)
    {
        Vector3 targetPos = boardSpaces[newPosition].transform.position;
        targetPos.y += 0.6f;
        p.token.transform.position = targetPos;
    }

    // Log the move for debugging
    Debug.Log($"{p.playerName} moved from {oldPosition} to {newPosition} "
           + $"({boardSpaces[newPosition].spaceName})");

    UpdateHUD();

    // Hand off to ResolveSpace — implemented in Lesson 6
    // For now, just advance the turn.
    NextTurn();  // temporary — will be replaced in Lesson 7
}

// Temporary NextTurn stub — full implementation in Lesson 7
public void NextTurn()
{
    currentIndex = (currentIndex + 1) % activePlayers.Count;
    UpdateHUD();
    ShowMessage($"{activePlayers[currentIndex].playerName}'s turn. Roll the dice!");
}
Why call NextTurn() here for now? In the final game, MovePlayer() will call ResolveSpace() instead — which may pause the turn while waiting for the player to decide whether to buy a property. We'll swap this out in Lesson 6. For now, calling NextTurn() directly lets you test movement immediately without needing the full game loop.

Part 4 · UI Button Wiring (10 min)

Connecting the Roll Dice Button

The code is written — now you need a button in the Unity UI that calls StartTurn() when clicked. This is done through the Unity Event System, not code.

1
Create a Canvas (if not already present)

In the Hierarchy: right-click → UI → Canvas. Unity automatically adds an EventSystem — leave it. Set the Canvas Render Mode to Screen Space — Overlay.

2
Add a Button

Right-click the Canvas in the Hierarchy → UI → Button - TextMeshPro. Rename it RollDiceButton. In the Inspector, set the button's child Text to "🎲 Roll Dice". Position it in the bottom-centre of the screen using the Rect Transform anchors.

Anchor: Bottom Centre  |  Pos X: 0  |  Pos Y: 60  |  Width: 160  |  Height: 50
3
Wire the OnClick event

Select RollDiceButton → look for the On Click () section in the Button component. Click the + icon to add an event. Drag the GameManager GameObject from the Hierarchy into the None (Object) slot. From the function dropdown: select GameManager → StartTurn().

4
Add a Message TMP_Text

Right-click Canvas → UI → Text - TextMeshPro. Rename it MessageText. Position it above the button. Drag it into the messageText slot on the GameManager Inspector. This makes ShowMessage() display on screen.

Anchor: Bottom Stretch  |  Pos Y: 120  |  Height: 40  |  Alignment: Centre
5
Press Play and test

Hit Play. Click the Roll Dice button. You should see: the token jump to a new space, the Console log the roll and new position, and the MessageText update. Click again — the next player's token should move. Try clicking until a player wraps past Go.

Expected Console: "Rolled: 4 + 3 = 7" → "Nomad moved from 0 to 7 (City Rd)"
⚠ Token not moving? The most common cause is that p.token is null — you forgot to drag the player's token GameObject into the token field in the Player Inspector (Lesson 4, Step 4). Check each Player's Inspector and confirm the token slot is filled.

Extension Work

Tiered Tasks

Basic
  • Add a TextMeshPro label above the button showing the last dice result (e.g. "⚄ + ⚂ = 8") — update it inside RollDice()
  • Disable the Roll Dice button after it is clicked and re-enable it at the start of each new turn — prevents double-rolling
Intermediate
  • Display the individual die values separately in the UI (two TMP labels side-by-side) so players can verify the roll
  • Move the token using Vector3.Lerp over 0.5 seconds instead of an instant snap — use a Coroutine so the rest of the code waits for the animation to finish before calling NextTurn()
  • Add a small Y-axis bounce to the token when it arrives at its destination (scale up briefly then back to normal using a Coroutine)
Advanced
  • Implement step-by-step movement: instead of jumping directly to the final space, move the token one space at a time with a 0.15s pause between each step — this makes it visually clear exactly how many spaces were moved
  • Add a doubles mechanic: if both dice show the same value, the player rolls again immediately — implement this in StartTurn() using a loop, but limit to 3 consecutive doubles before passing the turn
  • Write a probability distribution function that runs 1000 simulated dice rolls and logs how often each total (2–12) appeared — compare to the theoretical probability table and write a reflection in your notebook

Common Errors

Token doesn't move
p.token is null — drag the token GameObject into the Player Inspector's token field.

IndexOutOfRangeException
Your board has fewer spaces than expected — check boardSpaces.Length in the Console using Debug.Log(boardSpaces.Length) in Start().

Go salary fires every turn
The passedGo condition is wrong — double-check the newPosition <= oldPosition logic and confirm you're using && not ||.
