Part 1 · The Elimination Problem (10 min)

Why Turn Cycling Gets Complicated

In Lesson 5 we wrote a simple NextTurn() using modulo: (currentIndex + 1) % activePlayers.Count. This works perfectly for a fixed-size player list. But when a player is eliminated and removed from the list, two problems appear.

⚠ Problem 1 — Index skips a player

If Player B (index 1) is eliminated and removed, what was Player C (index 2) becomes index 1. If currentIndex was 1 when elimination happened, the next currentIndex + 1 skips to index 2 — Player C never gets a turn.

⚠ Problem 2 — Index out of range

If the last player in the list (index 2 of 3) is eliminated, activePlayers.Count drops to 2. currentIndex is now 2 — but the list only has indices 0 and 1. The next modulo wraps correctly, but we must clamp first.

The fix — clamp the index after removal

After removing a player, we clamp currentIndex so it stays within the new list bounds. Then, the call to NextTurn() handles normal advancement from there. Use the simulator below to see this in action.

Method 1 — Full turn cycling (replaces L5 stub) GameManager.cs 
// Advances to the next active player.
// Must handle the case where the list shrank after an elimination.
public void NextTurn()
{
    if (gameOver) return;
    if (activePlayers.Count == 0) return;

    // Clamp currentIndex in case a removal just shrank the list
    if (currentIndex >= activePlayers.Count)
        currentIndex = 0;

    // Advance to next player using modulo wrap
    currentIndex = (currentIndex + 1) % activePlayers.Count;

    UpdateHUD();
    ShowMessage($"{activePlayers[currentIndex].playerName}'s turn. Roll the dice!");
}
Method 2 — Bankruptcy check (replaces L6 stub) GameManager.cs 
// Called by ChargeRent() and ResolveSpace(Tax) when money drops below 0.
private void CheckBankruptcy(Player p)
{
    if (p.money < 0)
    {
        ShowMessage($"{p.playerName} is bankrupt with ${p.money}! Eliminated.");
        Debug.Log($"{p.playerName} eliminated. Remaining players: {activePlayers.Count - 1}");
        EliminatePlayer(p);
    }
    else
    {
        // Player survived — continue normally
        NextTurn();
    }
}
Method 3 — Remove a bankrupt player GameManager.cs 
private void EliminatePlayer(Player p)
{
    p.isActive = false;

    // Release all properties owned by this player back to the market
    foreach (BoardSpace space in p.ownedProperties)
        space.owner = null;
    p.ownedProperties.Clear();

    // Hide the token in the scene
    if (p.token != null)
        p.token.SetActive(false);

    // Get the index BEFORE removing (needed for clamp logic in NextTurn)
    int eliminatedIndex = activePlayers.IndexOf(p);
    activePlayers.Remove(p);

    // If the eliminated player came before or at currentIndex,
    // step currentIndex back so NextTurn advances to the correct player
    if (eliminatedIndex <= currentIndex && currentIndex > 0)
        currentIndex--;

    // Check if the game is now over
    if (activePlayers.Count <= 1)
        GameOver();
    else
        NextTurn();
}
Method 4 — End the game and show the winner GameManager.cs 
private void GameOver()
{
    gameOver = true;

    Player winner = activePlayers.Count > 0
        ? activePlayers[0]
        : null;

    string winnerName = winner != null ? winner.playerName : "No one";
    string winnerMoney = winner != null ? $"${winner.money}" : "$0";

    Debug.Log($"GAME OVER — Winner: {winnerName} with {winnerMoney}");

    // Show the Game Over panel
    if (gameOverPanel != null)
        gameOverPanel.SetActive(true);

    // Populate the winner text fields
    if (winnerNameText  != null) winnerNameText.text  = winnerName + " wins!";
    if (winnerMoneyText != null) winnerMoneyText.text = winnerMoney;
    if (winnerPropsText != null && winner != null)
        winnerPropsText.text = winner.ownedProperties.Count + " properties";

    // Disable the Roll Dice button — game is over
    if (rollDiceButton != null)
        rollDiceButton.interactable = false;
}

Common Errors

Player skipped after elimination
Index correction in EliminatePlayer() is wrong — re-check the eliminatedIndex <= currentIndex condition.

IndexOutOfRange after elimination
currentIndex not clamped before NextTurn() — confirm the clamp is at the top of NextTurn().

Game Over panel doesn't appear
gameOverPanel not dragged into GameManager Inspector, or the panel was left active (not set inactive in Start()).
