CT5-OPL-01

Algorithms and programming

Designs, produces and evaluates algorithms and implements them in a general-purpose and/or object-oriented programming language.

All content areas

Key content

  • Design, trace and represent algorithms using pseudocode, flowcharts and desk-checking techniques before writing code
  • Implement algorithms using sequence, selection and iteration in a general-purpose programming language, progressing to object-oriented approaches
  • Apply OOP principles — classes, objects, methods, attributes and inheritance — to model real-world systems and organise code
  • Use core programming constructs including variables, arrays, functions, parameters, return values and event handlers
  • Debug and test code systematically, distinguishing between syntax, runtime and logic errors, and using test cases to verify correctness
CT5-DPM-01

Design, produce and manage

Designs, produces and manages digital solutions using the design and production framework across all phases of a project.

All content areas

Key content

  • Apply the design and production framework — Identifying & Defining → Researching & Planning → Producing & Implementing → Testing & Evaluating — across all project work
  • Maintain a project notebook that documents design decisions, iterations, testing results and honest reflections throughout the project lifecycle
  • Develop functional and non-functional requirements by analysing user needs, stakeholder constraints and real-world context
  • Produce annotated design documentation — including wireframes, storyboards, flowcharts, data dictionaries, pseudocode and Gantt charts
  • Use project management strategies to plan tasks, allocate time, manage dependencies and adjust plans in response to testing and feedback
CT5-SOL-01

Software solutions

Designs and develops software solutions to meet identified needs using appropriate programming techniques, data structures and design patterns.

Creating games Developing apps and web software

Key content

  • Investigate and annotate existing software solutions to understand structure, patterns and programming approaches before beginning your own
  • Develop solutions that address identified real-world problems, considering usability, accessibility and the needs of diverse users
  • Select and apply appropriate data structures — arrays, lists, dictionaries, queues — based on the operations needed by the solution
  • Implement software security measures including input validation, error handling, data sanitisation and safe storage of user data
  • Produce well-structured, consistently named, commented and maintainable code that follows professional conventions
CT5-THI-01

Thinking skills

Applies computational, design and systems thinking when solving problems and producing digital solutions.

All content areas

Key content

  • Decompose complex problems into manageable sub-problems, and identify patterns and reusable solutions across different contexts
  • Apply abstraction to model only the essential features of a system, hiding unnecessary complexity from the user and other components
  • Design algorithms that anticipate edge cases, invalid inputs and error states before implementation
  • Apply systems thinking to understand how components interact — and what happens elsewhere when one part changes or fails
  • Use design thinking to keep the human user experience central throughout every phase of development, from research to evaluation
CT5-COL-01

Collaboration

Collaborates with others to plan, produce and evaluate digital solutions, communicating effectively across technical and non-technical contexts.

Creating games Developing apps and web software

Key content

  • Work collaboratively to plan, design and produce software solutions, defining roles, responsibilities and contribution expectations from the outset
  • Use version control and collaborative development tools to manage shared codebases, track changes and resolve conflicts
  • Give and receive constructive feedback during peer review and code review processes, focusing on evidence and improvement
  • Communicate technical decisions clearly to both technical and non-technical stakeholders, using appropriate formats and language
  • Contribute to a positive, inclusive team culture that values diverse perspectives, acknowledges contributions and manages conflict respectfully
CT5-EVL-01

Testing and evaluation

Tests and evaluates solutions against predetermined functional and non-functional requirements, considering social, ethical and cybersecurity dimensions.

All content areas

Key content

  • Develop test cases — including normal, boundary and error conditions — prior to building solutions, and use them to drive development
  • Conduct systematic testing, documenting actual vs expected outputs, identifying bugs and tracing them to their source in the code
  • Evaluate the quality of solutions against functional requirements, performance, security, accessibility and code maintainability criteria
  • Assess the social, ethical and environmental implications of software solutions for diverse communities, including those with disability
  • Communicate evaluation findings with evidence, proposing and implementing targeted improvements to address identified weaknesses
