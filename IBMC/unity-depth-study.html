01 — Overview

The Brief

You've been handed a Unity project with a working third-person character controller and a starter environment. Your job is to turn it into a complete, playable game — with a clear game loop, level design, and at least one extension of your own choice.

The core mechanic is already working: players move through a 3D world and collect coins. The UI updates in real time, the game manager tracks your score, and the coins rotate and disappear on collection. What happens next is up to you.

🎮
GameManager.cs

The central brain. Tracks total coins collected. Persists across scenes using DontDestroyOnLoad. Uses the Singleton pattern so any script can reach it.

03 — Scripts

Your Starter Code, Explained

Read these carefully. Every line matters. The annotations explain why the code is written this way, not just what it does.

GameManager.cs Singleton Pattern 
using UnityEngine;

public class GameManager : MonoBehaviour
{
    // The Singleton instance — one GameManager accessible everywhere
    public static GameManager Instance;

    private int totalCoins = 0;

    private void Awake()
    {
        if (Instance == null)
        {
            Instance = this;
            DontDestroyOnLoad(gameObject); // survive scene loads
        }
        else
        {
            Destroy(gameObject); // destroy duplicates
        }
    }

    public void UpdateCoinCount(int newAmount)
    {
        totalCoins = newAmount;
        UIManager.Instance.UpdateCoinUI(totalCoins);
    }

    public int GetCoinCount()
    {
        return totalCoins;
    }
}
🔑 Singleton Pattern

public static GameManager Instance is a static reference — it belongs to the class, not any specific object. This means any script can call GameManager.Instance.UpdateCoinCount() from anywhere in the project without needing a direct reference. The Awake() check ensures only one GameManager ever exists — if a second one appears (e.g. when loading a new scene), it destroys itself immediately.

🧩 Extension Point

This is where you add a win condition. Add a field public int totalCoinGoal = 20; and at the end of UpdateCoinCount(), check if (totalCoins >= totalCoinGoal) { CheckWin(); }. You can also add a timer here using Time.deltaTime in Update().

UIManager.cs TextMeshPro 
using UnityEngine;
using TMPro;

public class UIManager : MonoBehaviour
{
    public static UIManager Instance;

    [Header("UI Elements")]
    public TMP_Text coinText;

    private void Awake()
    {
        if (Instance == null) Instance = this;
        else Destroy(gameObject);
    }

    private void Start()
    {
        // Read from GameManager on scene start to sync display
        UpdateCoinUI(GameManager.Instance.GetCoinCount());
    }

    public void UpdateCoinUI(int amount)
    {
        coinText.text = "Coins: " + amount;
    }
}
📺 Extending the HUD

Add more TMP_Text fields for a timer, a coin goal display ("Coins: 12 / 20"), or a health bar. Create methods like UpdateTimerUI(float seconds) and call them from GameManager. All the wiring is the same pattern — Singleton, public method, update the text field.

PlayerCoinCollector.cs Component Bridge 
using UnityEngine;

public class PlayerCoinCollector : MonoBehaviour
{
    public int currentCoins = 0;

    public void AddCoins(int amount)
    {
        currentCoins += amount;

        // Pass the new total up to the GameManager
        GameManager.Instance.UpdateCoinCount(currentCoins);
    }
}
🧑 Why a separate PlayerCoinCollector?

Separation of concerns. The GameManager tracks the global game state; the PlayerCoinCollector tracks this player's coin count. In a multiplayer game you'd have two collectors feeding one GameManager. It also means the Coin doesn't need to know anything about the GameManager directly — it just tells the collector, which handles the rest.

coin.cs Trigger · Audio · VFX 
using UnityEngine;

public class Coin : MonoBehaviour
{
    [Header("Coin Settings")]
    public float rotationSpeed = 90f;
    public int coinValue = 1;

    [Header("Effects")]
    public AudioClip pickupSound;
    public GameObject pickupVFX;

    private void Update()
    {
        transform.Rotate(Vector3.up * rotationSpeed * Time.deltaTime);
    }

    private void OnTriggerEnter(Collider other)
    {
        if (other.CompareTag("Player"))
        {
            PlayerCoinCollector collector =
                other.GetComponent<PlayerCoinCollector>();

            if (collector != null)
                collector.AddCoins(coinValue);

            if (pickupSound)
                AudioSource.PlayClipAtPoint(pickupSound, transform.position);

            if (pickupVFX)
                Instantiate(pickupVFX, transform.position, Quaternion.identity);

            Destroy(gameObject);
        }
    }
}
🪙 Making Other Collectibles

Duplicate this script and rename it PowerUp.cs, Star.cs, or Key.cs. Change what happens in OnTriggerEnter — instead of calling AddCoins(), call a new method like AddHealth(), ActivateSpeedBoost(), or UnlockDoor(). Set coinValue to a higher number for a "gold coin" variant worth 5 points.

04 — Unity Tips

Things to Know Before You Start

🏷️

Tags Matter

The Coin script uses CompareTag("Player"). If your player object isn't tagged "Player" in the Inspector, coins will never register. Check this first if collection isn't working.

⚡

Is Trigger = On

The Coin's Collider must have "Is Trigger" checked. If it's a solid collider, OnTriggerEnter never fires — it'll be a wall instead of a collectible.

Inspector → Collider → ✓ Is Trigger
🎲

Prefabs are Your Friend

Turn your coin into a Prefab by dragging it from the Hierarchy into the Project panel. Then you can place dozens of coins quickly and edit them all at once.

Drag from Hierarchy → Project
📦

ProBuilder for Level Design

Window → Package Manager → ProBuilder. Lets you create custom geometry — platforms, ramps, rooms — directly in Unity without needing Blender.

Window → Package Manager
🔊

Free Audio Assets

Unity Asset Store has free audio packs. Freesound.org has CC-licensed clips. Drag an AudioClip into the pickupSound field on the Coin's Inspector — no code needed.

🐛

Use the Console

Window → General → Console. Read every red error message — Unity tells you the file and line number. A NullReferenceException almost always means a public field is empty in the Inspector.

Ctrl + Shift + C
▶️

Test Constantly

Hit Play after every change. Don't write 20 lines of code and then test — test every 5 minutes. Changes made in Play Mode are lost when you stop — use Debug.Log() to inspect values.

Ctrl + P to Play/Stop
💾

Save Often

Unity doesn't auto-save scenes. Use File → Save Scene frequently. If Unity crashes without saving, the last hour of level design is gone. Save before any major change.

Ctrl + S every 10 min
05 — Extensions

Make it Your Own

Choose at least one extension. Simpler done > complex broken. Read the description and pick something you'll actually finish in an hour.

Easy — ~30 min

⏱ Countdown Timer

Add a timer to the GameManager. When it hits zero, the player loses. Display it on the HUD.

  • Add float timeRemaining to GameManager
  • Decrement in Update() using Time.deltaTime
  • Call UIManager.Instance.UpdateTimerUI()
  • Trigger Game Over when it hits zero
Easy — ~30 min

💛 Gold Coins (Multi-Value)

Create a second coin prefab worth 5 points. Change the colour and scale. Set coinValue = 5 in the Inspector — no code changes needed.

  • Duplicate the Coin prefab
  • Change material colour to gold
  • Set coinValue to 5
  • Place sparingly in hard-to-reach spots
Medium — ~45 min

❤️ Health System

Add a health variable to the player. Create a hazard (lava, spikes) that reduces health on contact. If health hits zero, trigger Game Over.

  • Add int health = 3 to PlayerCoinCollector
  • Create a Hazard.cs script (copy Coin, change to TakeDamage)
  • Display hearts or a number in the HUD
  • Add a brief invincibility period after taking damage
Medium — ~45 min

⚡ Speed Power-Up

A collectible that temporarily boosts movement speed. Requires communicating with the character controller script.

  • Create SpeedBoost.cs (based on Coin)
  • Get the character controller component on pickup
  • Increase its speed variable for 5 seconds
  • Use a Coroutine to reset it after the duration
Hard — ~60 min

🚪 Scene Loader / Next Level

Add a second scene (level 2). When the player collects all coins, load the next scene. The GameManager persists coin count across scenes automatically.

  • File → Build Settings → add Scene 2
  • Add SceneManager.LoadScene() to win condition
  • Place coins in Scene 2
  • Show running total between levels on a transition screen
Hard — ~60 min

👾 Simple Enemy

A patrolling enemy that moves between waypoints. If it touches the player, reduce health or trigger a respawn.

  • Create an Enemy object with a simple model
  • Write Enemy.cs with a Transform array of waypoints
  • Move between them with Vector3.MoveTowards
  • Use OnTriggerEnter to detect player contact
Stretch — 60+ min

🏆 Persistent Leaderboard

Save the player's score using PlayerPrefs and display a local high score on the main menu or game over screen.

  • PlayerPrefs.SetInt("HighScore", score)
  • Load with PlayerPrefs.GetInt("HighScore")
  • Compare after each run and update if beaten
  • Display on a dedicated UI panel
Stretch — 60+ min

🌟 Animated Coin Trail

Coins "fly" toward the player UI on collection instead of just disappearing. Uses a Coroutine to lerp the coin object toward the HUD position.

  • Don't Destroy immediately — start a Coroutine
  • Move coin toward a UI target position in world space
  • Scale down and fade alpha over 0.5 seconds
  • Add coin count only after animation completes
06 — World Building

Level Design Principles

A good level teaches players without a tutorial. The arrangement of your coins, the shape of your terrain, and the placement of obstacles should naturally guide the player and reward exploration.

✅ Do This

  • Place a cluster of easy coins near the start to teach the mechanic
  • Use height variation — coins on raised platforms reward platforming skill
  • Create a clear visual language — coins glow, hazards look dangerous
  • Leave breathing room — don't pack everything into one corner
  • Give the player a moment to survey the level from the start position
  • Playtest with someone who hasn't seen the level

❌ Avoid This

  • Placing all coins in a straight line — no decision-making for the player
  • Invisible walls — use visible geometry to block paths
  • A coin that requires pixel-perfect jumping to reach (frustrating, not fun)
  • A completely flat plane with no landmarks — players get lost
  • Too many coins — dilutes the reward feeling of each collection
  • Finishing the level before testing it yourself in Play Mode
💡 The Rule of Three

For every mechanic in your game, design three examples in your level: one that's easy (shows players how it works), one that's moderate (makes them think), and one that's hard (rewards mastery). This applies to coin placement, obstacles, and any collectibles you add.

07 — Assessment

How Your Demo is Evaluated

Your game is assessed as a playable demo. It must be launchable and completable by someone who has never seen it, without crashing.

Coin Collection All provided scripts working correctly, coins collect reliably, HUD updates instantly, no console errors Collection works but has occasional bugs or console warnings Collection broken — scripts not connected or tag/trigger misconfigured / 4
