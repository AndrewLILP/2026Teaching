Generalising from Arithmetic

The deepest idea in algebra isn't a procedure — it's a claim: that rules which hold for specific numbers hold for all numbers. This section asks you to examine that claim, justify it, and know its limits.

💡 Reasoning — Primary focus 🗣 Communicating 🔗 Understanding & Fluency
Central Question

"How do we know a rule works for all values — not just the ones we tried?"

📌

Task 2 💡 Reasoning

Always, Sometimes, or Never?

For each statement below, decide whether it is always true, sometimes true, or never true. Select your answer, then reveal the reasoning. Being wrong and understanding why is the point.

📌
Before revealing each answer, commit to your choice. For "sometimes" answers, always ask yourself: for what values is it true? For what values is it false? That's where the mathematics lives.
a − b = b − a
Sometimes true

Subtraction is not commutative. Reversing the order changes the sign of the result: a − b = −(b − a). The statement is only true when a − b = 0, meaning a = b.

Counterexample: a = 8, b = 3 → 8 − 3 = 5, but 3 − 8 = −5. Not equal.
When it is true: a = 6, b = 6 → 6 − 6 = 0 = 6 − 6. ✓

This is a critical misconception to avoid when simplifying algebraic expressions. Order matters for subtraction.

ab = ba
Always true

Multiplication is commutative. The commutative property of multiplication states that for any numbers a and b, a × b = b × a — and since ab is algebraic notation for a × b, we have ab = ba.

Try it: a = 5, b = 3 → ab = 15, ba = 15. ✓ || a = −4, b = 7 → ab = −28, ba = −28. ✓

This is why we can rearrange factors freely when simplifying expressions like 4x × 3y.

3p + 2q = 5pq
Never true (as a general identity)

3p + 2q is an addition of unlike terms. 5pq is a product. These are fundamentally different structures and can never be equal for all values of p and q.

Counterexample: p = 1, q = 1 → 3(1) + 2(1) = 5, but 5(1)(1) = 5. Wait — they are equal here!

But: p = 2, q = 1 → 3(2) + 2(1) = 8, but 5(2)(1) = 10. Not equal.

The key error here is adding unlike terms and producing a product. Addition and multiplication produce different results. This statement might be true for a specific pair of values but is not a valid algebraic identity — the expressions are not equivalent.

(a + b) + c = a + (b + c)
Always true

This is the associative property of addition. It states that when adding three or more numbers, the grouping (which pair you add first) does not affect the result. This holds for all real numbers and therefore for all algebraic expressions.

a = 2, b = 5, c = 8: (2 + 5) + 8 = 7 + 8 = 15 = 2 + 13 = 2 + (5 + 8). ✓

This property means we can remove or rearrange brackets in additions without changing the result — which is fundamental to simplifying algebraic expressions. Note: the same property holds for multiplication.

5x + 5y = 5(x + y)
Always true

This is the distributive law — but notice what direction it's being used here. We're going from the expanded form (5x + 5y) to the factorised form (5(x + y)). This is exactly what factorising does.

x = 3, y = 4: 5(3) + 5(4) = 15 + 20 = 35 = 5(3 + 4) = 5(7) = 35. ✓

Look ahead: This statement connects what you've learned about simplifying to what you're about to learn in factorising (Section 5). The ability to "factor out" a common factor is the distributive law running in reverse.

4x² + 3x² = 7x⁴
Never true (as a general identity)

Two errors in one statement. The terms 4x² and 3x² are like terms (both contain x²), so they can be combined — but only by adding the coefficients, not the powers.

The correct simplification is: 4x² + 3x² = 7x² (not 7x⁴).

x = 2: 4(4) + 3(4) = 16 + 12 = 28 = 7(4) = 7x². ✓
But 7x⁴ = 7(16) = 112 ≠ 28. The power is never added when collecting like terms.

Adding the exponents is what happens when you multiply powers of the same base (e.g., x² × x³ = x⁵), not when you add like terms. This is one of the most common errors in algebra.

Task 3 🗣 Communicating 💡 Reasoning

Convince Me

A mathematical argument is more than saying something is wrong — you need to show why, using mathematical structure, examples, and precise language. Write your argument before revealing a model response.

Convince Me A
A student says: "I can simplify 5a + 3b, because both terms have letters in them. So 5a + 3b = 8ab."

Write a mathematical argument that explains clearly why this student is wrong. Your argument should identify the specific error, explain what like terms actually means, and show why the correct simplification is that the expression cannot be simplified further.

✅ One possible argument

The student has made two related errors. First, they have treated 5a and 3b as like terms because both expressions contain a letter. But like terms must contain exactly the same pronumeral(s) raised to exactly the same powers. The term 5a contains the pronumeral a; the term 3b contains the pronumeral b. Since a and b are different pronumerals representing potentially different values, these are unlike terms and cannot be combined by addition.

Second, even if the student could add the terms, the result 8ab is incorrect: it would imply multiplication (8 times a times b), not addition. Adding expressions never produces a product.

A counterexample makes this concrete: let a = 2 and b = 5. Then 5a + 3b = 10 + 15 = 25. But 8ab = 8 × 2 × 5 = 80. These are clearly different values, so the claimed simplification is false. The expression 5a + 3b is already fully simplified — it cannot be written with fewer terms.

Convince Me B
A student says: "When I multiply algebraic terms, the order I write them in doesn't matter. 4 × m × 3 × n is the same as 3 × 4 × m × n."

The student is correct, but their explanation is incomplete. Write a mathematical argument that supports their claim and names the exact property (or properties) of arithmetic that justify it. Your argument should also identify the limits of this claim — are there situations in algebra where order does matter?

✅ One possible argument

The student is correct. The justification is the commutative property of multiplication, which states that for any two numbers a × b = b × a. This can be applied repeatedly when multiplying several terms together: since each swap of adjacent factors is justified by the commutative property, the factors can be rearranged in any order without changing the product.

Concretely: 4 × m × 3 × n = 4 × 3 × m × n = 12mn. Each step uses the commutative property to move the 3 to sit next to the 4. The associative property of multiplication then allows us to group (4 × 3) and evaluate it as 12.

However, the claim has limits. Order matters critically for subtraction and division: a − b ≠ b − a (in general) and a ÷ b ≠ b ÷ a (in general). So when simplifying expressions involving subtraction or division, terms and factors cannot be freely reordered. For example, 12 ÷ 3 = 4, but 3 ÷ 12 = 0.25.

Task 4 💡 Reasoning 🔗 Fluency

The Boundary Question

We can simplify 5a + 3a but not 5a + 3b. This task unpacks exactly why — and the answer is more interesting than a rule to memorise.

🔎 Follow the reasoning — step by step

Work through each step below. Don't skip ahead — each step builds the understanding that makes the final insight click.

1
Try some numbers for 5a + 3a

Substitute values into the expression 5a + 3a and into 8a. Do they always match?

a = 2:   5(2) + 3(2) = 10 + 6 = 16
→
8(2) = 16 ✓
a = 7:   5(7) + 3(7) = 35 + 21 = 56
→
8(7) = 56 ✓
a = −3:   5(−3) + 3(−3) = −15 + (−9) = −24
→
8(−3) = −24 ✓
2
The "units" argument — what are these terms actually counting?

5a means "5 lots of a". 3a means "3 lots of a". If you have 5 lots of the same thing and then 3 more lots of the same thing, you have 8 lots of that thing altogether.

The pronumeral a is the unit. Both terms are measuring in the same unit, so they can be combined — exactly like adding 5 apples + 3 apples = 8 apples.

💡
Like terms can be combined because they express the same mathematical unit. The coefficient tells you how many of that unit you have.
3
Substitute into 5a + 3b — does 8 times anything equal the result?

Let's try a = 2 and b = 5.

5(2) + 3(5) = 10 + 15 = 25
→
8a = 8(2) = 16 ≠ 25 ✗
5(2) + 3(5) = 10 + 15 = 25
→
8b = 8(5) = 40 ≠ 25 ✗

There is no single pronumeral you can write after the 8 that will always match 5a + 3b, because the result depends on two independent values that can vary separately.

4
The generalisation: why "like" matters

5a and 3b measure in different units — "lots of a" and "lots of b". Since a and b can take different values independently, you cannot combine them into a single term.

This is exactly like being unable to add "5 apples + 3 oranges = 8 ?" — 8 what? There is no single meaningful unit for the sum.

⭐
The definition of like terms (restated with understanding): Terms are "like" when they express the same unit — that is, they contain exactly the same pronumeral(s) raised to exactly the same power(s). The coefficients tell you how many of that unit each term represents, and those coefficients are what you add or subtract.
✏️ Consolidate your understanding

Without looking back, write in your own words why 7x² + 2x² can be simplified, but 7x² + 2x cannot. Use the "units" idea in your explanation.

Task 5 🔗 Fluency 💡 Reasoning

Two Methods

Two students simplify the same expression differently. Both reach the correct answer — but the question is what mathematical property is being used at each step.

🔢 Simplify:  4m × 3m

Study both methods carefully before answering the questions below.

📘 Method A — Group, then simplify
Identify the numerical and algebraic factors separately
4m × 3m = (4 × 3) × (m × m)
Multiply the numerical factors
= 12 × (m × m)
Apply index notation
= 12m²
📗 Method B — Reorder all factors
Write out every individual factor
4m × 3m = 4 × m × 3 × m
Reorder: group numbers together, then letters
= 4 × 3 × m × m
Multiply numbers, apply index notation
= 12 × m² = 12m²
a
In Method B, Step 2 rearranges the order of the four factors. Name the exact arithmetic property that makes this step valid. Why does this property apply here but not to subtraction?
b
Method A skips directly to grouping numbers and letters. What assumption is it making — and what property guarantees that assumption is valid?
c
Both methods produce 12m². Try the expression 5x² × 2xy. Which method would you choose and why? Show your working, naming any properties you use.
d
Extension: A student simplifies 4m × 3m and writes 12m as their answer. Identify the error and write an explanation they would understand. What has the student forgotten about what m × m means?

🔴 Extension — These tasks require you to think about algebraic structure, generality, and the difference between an example and a proof.
1
Write an algebraic statement of the form "Expression A = Expression B" that is sometimes true — true for some values of the variable(s) but not all. Identify all values for which it is true, and all values for which it is false. Justify your answer.
2
The commutative property holds for addition and multiplication. Does a "commutative-like" property hold for exponentiation? That is, is it always true that aᵇ = bᵃ? Investigate with examples. If it's not always true, find all pairs of positive integers for which it is true.
Start by trying small values: 2³ vs 3², 2⁴ vs 4²...
3
You've seen that checking examples doesn't prove a property holds for all values. But it can disprove it — one counterexample is enough to show something is false.
