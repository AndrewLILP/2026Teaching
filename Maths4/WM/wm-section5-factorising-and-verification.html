Factorising and the Art of Verification

The syllabus asks you to verify your factorisations by expansion — but what does passing that check actually tell you? This section examines what verification proves, what it doesn't, and why the difference matters.

💡 Reasoning — Primary focus 🧩 Problem Solving 🗣 Communicating
Central Question

"How does verifying by expansion give you certainty — and what, exactly, are you certain of?"

Task 1 🔗 Fluency 💡 Reasoning

Inverses — The Structural Insight

Factorising isn't a new operation — it's the reverse of one you already know. Understanding this relationship is the foundation for understanding why verification works.

🔄 Operations and their inverses

Every operation in mathematics has an inverse — an operation that undoes it. We use these inverse relationships to check our work all the time.

Addition ↔ Subtraction
5 + 3 = 8
Check: 8 − 3 = 5 ✓
Multiplication ↔ Division
4 × 6 = 24
Check: 24 ÷ 6 = 4 ✓
Expanding ↔ Factorising
3(x + 4) → 3x + 12
Check: 3x + 12 → 3(x + 4) ✓

🗺️ The two-way relationship

Factorised form
4(3x + 2)
→ Expand (Distributive Law)
← Factorise (find HCF)
Expanded form
12x + 8

Starting from the factorised form and expanding gives the expanded form. Starting from the expanded form and factorising should give back the factorised form. When these round trips are consistent, the two forms are equivalent — they have the same value for every possible value of x.

Respond to these
✏️ Part A

In the examples above, the "check" for addition is to subtract. The "check" for multiplication is to divide. Using this pattern: what is the natural "check" for a factorisation — and why does it work?

✏️ Part B

If you expand a factorised expression and get back the original, you've shown the two forms are equivalent. What would it mean if you didn't get back the original? What would that tell you about your factorisation?

✏️ Part C — harder question

Consider: you perform a factorisation, expand to check, and get the original expression back. A friend says: "That proves you've factorised it correctly." Another friend says: "That only proves the two forms are equal — it doesn't prove the factorisation is complete." Who is right? Explain.

Task 2 💡 Reasoning

What Does Verification Actually Prove?

Three students factorise the same expression and all pass the verification test. But not all of them are finished.

🧪 Three factorisations of 12x + 8

Read each student's work carefully. For each, the verification step is shown. Your job is to evaluate not just whether the verification passes, but whether the factorisation is complete.

Student Alpha

Student Alpha's working
1
12x + 8 = 2(6x + 4)
Factor out 2
2
Verify: 2(6x + 4) = 12x + 8 ✓
Expand to check
Verification: PASS The expanded form matches the original.
⚠️ Factorisation incomplete. The factor 2 is a common factor of 12 and 8, but it is not the HCF. HCF(12, 8) = 4. The expression inside the brackets, 6x + 4, still contains a common factor of 2. Fully factorised: 4(3x + 2).
✏️ Discuss

Verification passed — so why is the work incomplete? What does the syllabus mean by "fully factorised"?

Student Beta

Student Beta's working
1
HCF(12, 8) = 4
Find HCF of the coefficients
2
12x + 8 = 4(3x + 2)
Factor out HCF
3
Verify: 4(3x + 2) = 12x + 8 ✓
Expand to check
Verification: PASS The expanded form matches the original.
✅ Factorisation complete. The HCF of 12 and 8 is 4, and 3x + 2 has no common factor. This is the fully factorised form.

Student Gamma

Student Gamma's working
1
12x + 8 = 1(12x + 8)
Factor out 1
2
Verify: 1(12x + 8) = 12x + 8 ✓
Expand to check
Verification: PASS The expanded form matches the original.
⚠️ Not a useful factorisation — and not complete. Factoring out 1 is always valid (trivially), but it changes nothing. This is mathematically true but doesn't demonstrate any factorisation skill. The HCF is 4, not 1.
✏️ The key insight

All three students passed the verification test. Only Student Beta's work is complete. What does this tell you about what the verification test alone can guarantee?

⭐
The central distinction — hold this in mind for the rest of the section: Verification by expansion proves that a factorised form is equivalent to the original — both expressions have the same value for every value of the variable. It does not prove the factorisation is complete. A factorisation is complete only when the HCF has been factored out — meaning the expression inside the brackets cannot itself be factorised further.

Task 3 💡 Reasoning 🧩 Problem Solving

Sorting Factorisations

For the expression below, four students each write a different factorisation. For each one: decide whether the verification would pass, then decide whether the factorisation is complete. Reveal to check.

Factorise this expression
18ab + 24a
💡
Before revealing each verdict: for each factorisation, ask yourself two separate questions. (1) If I expand this, do I get 18ab + 24a back? (2) Is there any common factor still hiding inside the brackets?
Student 1
18ab + 24a = 6(3ab + 4a)
✓ Verification passes ⚠ Incomplete
6(3ab + 4a) = 18ab + 24a ✓
6 is a common factor of 18 and 24, so the verification passes. However, the expression inside the brackets, 3ab + 4a, still contains a common factor of a. This should have been factored out as well. The HCF is 6a, not 6. Fully factorised: 6a(3b + 4).
Student 2
18ab + 24a = 6a(3b + 4)
✓ Verification passes ✓ Complete
6a(3b + 4) = 18ab + 24a ✓
The HCF of 18ab and 24a is 6a (HCF of 18 and 24 is 6; common algebraic factor is a). The expression inside the brackets, 3b + 4, has no common factor. This is the fully factorised form.
Student 3
18ab + 24a = 3a(6b + 8)
✓ Verification passes ⚠ Incomplete
3a(6b + 8) = 18ab + 24a ✓
3a is a common factor, so the verification passes. But the expression inside the brackets, 6b + 8, still has a common factor of 2. The factor taken out (3a) is not the HCF. Fully factorised: 6a(3b + 4).
Student 4
18ab + 24a = 6a(3b + 4a)
✗ Verification fails ✗ Incorrect
6a(3b + 4a) = 18ab + 24a² ≠ 18ab + 24a ✗
This factorisation is wrong. Expanding reveals the second term becomes 24a² (not 24a) — the student has written an extra a inside the brackets that shouldn't be there. When dividing 24a ÷ 6a, the result is 4, not 4a. The verification check catches this error immediately.
✏️ Synthesise

Three of the four factorisations passed the verification test. Only one was both correct and complete. Write a rule that tells you, after verification passes, how to check whether your factorisation is complete.

Task 4 🗣 Communicating 💡 Reasoning

Convince Me

Write mathematical arguments that show you understand the difference between what verification proves and what it doesn't. Commit to writing before revealing.

Convince Me A
A student says: "I factorised 20x + 15 as 5(4x + 3) and expanded to check — I got 20x + 15 back, so my factorisation must be correct and complete."

The student's factorisation is actually correct and complete in this case. But their reasoning about why it's complete is insufficient — the verification alone didn't tell them that. Write an argument that explains what the verification proved, what additional check confirms the factorisation is complete, and why this particular example happens to satisfy both.

✅ One possible argument

The student is correct that their factorisation is complete, but their justification is incomplete. Verification by expansion proves only that 5(4x + 3) and 20x + 15 are equivalent expressions — they have the same value for every value of x. This is a necessary condition for a correct factorisation, but not sufficient to confirm it is fully factorised.

To confirm the factorisation is complete, a separate check is required: examine the expression inside the brackets and confirm it cannot be factorised further. In this case, the expression inside the brackets is 4x + 3. The coefficients 4 and 3 share no common factor other than 1, so 4x + 3 cannot be simplified further. This confirms that 5 is the highest common factor of 20 and 15, which can also be verified directly: HCF(20, 15) = 5.

So the factorisation is complete — but this conclusion comes from checking the HCF, not from the verification test itself. A student who skipped the HCF check and wrote 5(4x + 3) might have written 5 by chance rather than by finding the true HCF. The verification would pass either way, but only the HCF check confirms completeness.

Convince Me B
A student says: "Verification is pointless — if I've done my factorisation correctly, it will obviously expand back to the original. It's just wasted steps."

The student has misunderstood the purpose of verification. Write a mathematical argument that explains the genuine value of the verification step — using a specific example where a plausible error in factorising would be caught immediately by expansion but might not be noticed by inspection alone.

✅ One possible argument

Verification is not pointless — it serves as a reliable mathematical check against errors that are easy to make and hard to catch by inspection. Factorising involves working backwards from an expression, which is inherently more error-prone than the direct calculation of expanding.

Consider factorising 15x² + 10x. A student might write 5x(3x + 2), which looks plausible. Without verification, this might pass unnoticed. But expanding reveals:

5x(3x + 2) = 15x² + 10x ✓

This is correct. Now suppose the student instead wrote 5x(3x + 10) by misreading the second term. Verification immediately shows:

5x(3x + 10) = 15x² + 50x ≠ 15x² + 10x ✗

The verification fails and alerts the student to the error. By inspection, 5x(3x + 10) might look reasonable — both factors appear plausible. But the expansion is unambiguous.

The value of verification is that it is decisive: the expanded form either matches or it doesn't. There is no ambiguity, no need to rely on whether the answer "looks right." It transforms a subjective check into an objective one, which is precisely what mathematical verification is for.

Task 5 💡 Reasoning 🧩 Problem Solving

Verify and Assess

For each factorisation below, decide: (A) would verification pass?, and (B) is the factorisation complete? Select your verdict, then reveal the full explanation.

📌
Two separate questions every time: "Does it expand back?" (verification) and "Is the bracket fully simplified?" (completeness). Get used to asking both — the first is a quick check, the second requires more thought.
1
8m + 12 = 2(4m + 6)
✓ Verification passes ⚠ Incomplete
2(4m + 6) = 8m + 12 ✓

HCF(8, 12) = 4, not 2. The expression inside the brackets, 4m + 6, has a common factor of 2. Fully factorised: 4(2m + 3). The bracket in the fully factorised form, (2m + 3), has no common factor — the integers 2 and 3 are coprime.

2
15y − 10 = 5(3y − 2)
✓ Verification passes ✓ Complete
5(3y − 2) = 15y − 10 ✓

HCF(15, 10) = 5. Inside the bracket, 3y − 2: the integers 3 and 2 are coprime (HCF = 1), so no further factorisation is possible. This is fully factorised.

3
6x² + 9x = 3(2x² + 3x)
✓ Verification passes ⚠ Incomplete
3(2x² + 3x) = 6x² + 9x ✓

The numerical factor 3 is correct, but the algebraic factor x has been missed. The expression inside the bracket, 2x² + 3x, still has a common factor of x. The HCF is 3x (not just 3). Fully factorised: 3x(2x + 3).

This is a very common error — students find the numerical HCF but forget to factor out the algebraic common factor. Always check: does each term in the original expression share a common pronumeral factor?

4
14pq + 21p = 7p(2q + 3)
✓ Verification passes ✓ Complete
7p(2q + 3) = 14pq + 21p ✓

HCF of 14 and 21 is 7. The common algebraic factor is p (both terms contain p, neither contains q in both). Inside the bracket: 2q + 3 has no common factor. This is fully factorised.

5
10a²b + 15ab² = 5ab(2a + 3b)
✓ Verification passes ✓ Complete
5ab(2a + 3b) = 10a²b + 15ab² ✓

HCF of 10 and 15 is 5. The common algebraic factor: both terms contain a and b. First term 10a²b has factors including ab; second term 15ab² has factors including ab. So algebraic HCF is ab. Combined HCF = 5ab. Inside the bracket: 2a + 3b — the integers 2 and 3 are coprime, and a and b are different pronumerals, so no further factorisation is possible. Complete.

6
12x + 8 = 4(3x + 3)
✗ Verification fails ✗ Incorrect
4(3x + 3) = 12x + 12 ≠ 12x + 8 ✗

The factorisation is wrong. Dividing 8 by 4 gives 2, not 3. The correct factorisation is 4(3x + 2). The verification check catches this immediately — the constant term 12 does not match the original constant 8. This is a case where the expansion is doing exactly what it should: exposing an arithmetic error in the factorising step.

Task 6 🧩 Problem Solving 🗣 Communicating

Create Your Own

Working backwards — constructing problems that meet specific conditions — is harder than solving given ones. These tasks ask you to design, not just solve.

🟢 Foundation — Start with the factorised form and work outwards. Verify each result by expanding.
1
Write an algebraic expression with exactly two terms that has an HCF of 5. Factorise it, then verify by expansion. Show every step.
Start: pick two multiples of 5, attach a pronumeral to one or both, combine.
2
You are given the fully factorised form 3(4x + 7). Without expanding first, write what you expect the expanded form to be. Then verify by expanding. Were you right?
3
A student factorisees an expression and gets 2(5x + 4). They expand to check and get 10x + 8. Write the original expression the student started with. Is their factorisation complete? How do you know?
🔵 Core — Design expressions and factorisations with specific properties. Always verify, and always check for completeness separately.
1
Write an expression with three terms that has a common algebraic factor of 4x. Factorise it fully and verify by expansion. Explain how you know 4x is the HCF.
Build it: pick three terms, each one a multiple of 4x, with different "remainders" inside.
2
Design a factorisation question where a student who factors out a partial factor (not the HCF) would still pass the verification test. Write the expression, the partial factorisation, the verification, and then the fully factorised form. Explain why verification passed but completeness was not achieved.
3
You are told that an expression, when fully factorised, gives 6ab(3a + 2b − 1). Write the expanded form of this expression. Then verify your expansion is correct by re-factorising it and confirming you get back to the factorised form.
🔴 Extension — These tasks require structural thinking about factorisation and what it means for two expressions to be equivalent.
1
Is it possible to fully factorise an expression in more than one way — that is, can two genuinely different factorised forms both be "complete"? Investigate with examples. If you believe the answer is no, write a mathematical argument for why the fully factorised form is unique. If you believe the answer is yes, give a counterexample.
Think about what "HCF" means, and whether it is unique for a given expression.
2
The verification step proves that the factorised and expanded forms are equivalent — they have the same value for every possible value of the variable. Write a general argument for why this is true: that is, if a(b + c) = ab + ac holds for all numbers, then factorising using this law always produces an equivalent expression. What assumption are you relying on?
3
Design a three-part problem of the form: (i) an expression that looks fully factorised but isn't — one more step is hidden; (ii) a factorisation that passes verification but has an error that only emerges when substituting a specific value of the variable; (iii) an expression with an algebraic HCF that most students would miss because they only look at the numerical coefficients. For each part, write the expression, the problematic factorisation, and its correction.
