# Digital Textbook Chapter Builder — Reference Guide

> **Before you start:** Always provide the **NESA outcome code** and the **full list of content dot points** from the syllabus for best results. The more precisely these match the syllabus wording, the more accurately the content, vocabulary, and examples will be targeted.
>
> Example prompt structure:
> ```
> Please make a webpage for this content:
>
> [Topic name]
> Outcome: MA4-EQU-C-01
>
> * Content dot point 1
> * Content dot point 2
> * Content dot point 3
> ```

---

## What gets built

A single, self-contained HTML file. No external dependencies beyond Google Fonts (Nunito). Works offline and in any browser — suitable for classroom use without reliable internet.

---

## Tab structure (7 tabs)

Each page is built with this standard tab layout. Tabs can be adapted to suit the topic, but this structure has proven effective for struggling students.

| Tab | Label | Purpose |
|-----|-------|---------|
| 1 | 📚 Key Words | Vocabulary pre-teaching before any content is introduced |
| 2 | 💡 Concepts | Core conceptual explanation and comparisons |
| 3 | 🔄 Visual / Interactive | Topic-specific visual model or interactive tool |
| 4 | ✏️ Worked Examples | Step-reveal I Do examples across all content points |
| 5 | 🔍 Error Detective | Spot-the-mistake activities with anonymous worked solutions |
| 6 | 📖 / 📐 Word / Formula Problems | Newman's framework or formula problem scaffolds |
| 7 | ✅ Practice | Tiered self-marking MCQ exercises |

Tab 3 has been customised per topic:
- **Equations (linear):** Balance scale SVG — interactive operations on both sides
- **Substitution:** Live substitution demo — type a value, watch it replace the pronumeral step by step
- **Quadratics:** Interactive parabola — slider moves y = c, shows ±√c intersection points

## Features included in every chapter

### Vocabulary (Tab 1)
- Six flip cards per chapter (tap/click to flip)
- Each card uses the **Frayer model**: definition, examples, non-examples, key idea
- Cards cover the essential terms for the topic before students encounter them in context
- Terms are chosen to address the most common language barriers for struggling students

### Concepts (Tab 2)
- Side-by-side comparisons where relevant (e.g. expression vs equation, solving vs verifying)
- Visual process strips for multi-step procedures
- Reference tables (e.g. common formulas with pronumeral meanings)
- An embedded quick-check MCQ to confirm understanding before moving to solving

### Visual / Interactive model (Tab 3)
- One interactive element per chapter, built as SVG or HTML/JS — no external libraries
- Designed to make the core conceptual question visible (why two solutions? what does balance mean? what is substitution doing?)
- Includes a key rules summary panel beneath the interactive

### Worked Examples (Tab 4)
- Five examples per chapter, spanning Foundation → Core → Extension
- **Step-reveal format:** one step shown at a time via "Show Next Step" button
- Each step shows: the action taken, the algebra, and a *why* explanation in italics
- Covers all content dot points across the five examples
- Examples use anonymous working (no named fictional students making errors)

### Error Detective (Tab 5)
- Two cases per chapter
- Anonymous worked solutions with exactly one deliberate error each
- Students guess the wrong line before clicking Reveal
- Reveal explains the specific misconception, gives the correct working, and names the common error pattern
- Errors are chosen to target the highest-frequency mistakes for that topic

### Word / Formula Problems (Tab 6)
- Three problems per chapter
- Uses the **Newman's Error Analysis framework** — five visible steps: Read → Define → Write → Solve → Answer
- Each step has a **sentence frame with blanks** for students to work into
- Full solution is hidden until student requests it
- Problems escalate in complexity: 1-step → 2-step → multi-variable or pronumerals both sides

### Practice (Tab 7)
- **Three tiers:** Foundation / Core / Extension (toggle switch)
- Four MCQs per tier (twelve questions total per chapter)
- Self-marking: correct answer turns green with ✅ explanation; incorrect turns red with ❌ redirect
- **Feedback causes thinking:** incorrect feedback doesn't just say "wrong" — it points to the next step or identifies the error type
- **Hint system:** each question has a collapsible hint (💡) that gives a nudge without giving the answer away
- Tier descriptions explain what each level focuses on

## Fraction rendering

All fractions use a CSS `.frac` class — not the slash character. This is critical for readability.

```html
<!-- Example: x/3 + 2 = 7 should be written as: -->
<span class="frac"><span class="num">x</span><span class="den">3</span></span> + 2 = 7
```

The CSS for this class:
```css
.frac { display:inline-flex; flex-direction:column; align-items:center; vertical-align:middle; line-height:1; margin:0 2px; }
.frac .num { border-bottom:1.5px solid currentColor; padding:0 3px 2px; font-size:.85em; }
.frac .den { padding:2px 3px 0; font-size:.85em; }
```

## Pedagogical principles embedded in the design

These come directly from the NSW syllabus project files and evidence-based planning advice:

- **Pre-teaching vocabulary** before content reduces cognitive load (language support document)
- **Frayer model** flip cards build definition + examples + non-examples simultaneously
- **I Do / We Do / You Do** maps to: Worked Examples → Error Detective / Word Problems → Practice
- **Scaffolding reduces progressively** — sentence frames in word problems, hints in practice, support decreasing tab by tab
- **Feedback causes thinking** — MCQ feedback redirects rather than just corrects (evidence-based planning advice)
- **Error Detective uses anonymous errors** — avoids the pastoral concern of named fictional students being associated with mistakes
- **Newman's framework** structures word problem access: decode → comprehend → transform → process → encode
- **Tiered practice** — Foundation / Core / Extension with a switching mechanism, visible within the same resource
- **Visual representations paired with symbolic** — every interactive tab provides a diagram-level entry point before algebraic working

## Fixes applied during this project (remember for future chapters)

- **Flip card height:** set to `280px` minimum; back face needs `overflow: auto` not `overflow: hidden`
- **Fractions:** replace all `x/3` style notation with `.frac` span markup — check especially in: step-math divs, MCQ question spans, balance model panels, and remember-card prose
- **Slash notation in JS feedback strings** (inside `onclick` attributes): cannot render HTML — use `÷` symbol or plain text instead
