Chapter 02 · Programming Fundamentals

Data & Algorithms

Every game is built on data — player health, inventory items, leaderboard scores, map coordinates. Before writing code, engineers must understand how computers store and represent that data. This chapter covers the number systems, data types, and data structures that underpin all software.

Section 2.1

Number Systems

Computers store everything — images, sounds, code, player data — as sequences of binary digits (bits): ones and zeros. As a software engineer you need to fluently read and convert between three number systems: binary (base 2), decimal (base 10), and hexadecimal (base 16).

How Positional Number Systems Work
In any positional number system, each digit's value depends on its position. Moving left, the column value multiplies by the base.

Decimal (base 10): columns are 1, 10, 100, 1000 …
Binary (base 2): columns are 1, 2, 4, 8, 16, 32, 64, 128 …
Hexadecimal (base 16): columns are 1, 16, 256, 4096 … using digits 0–9 then A–F

To convert binary to decimal, identify which bit columns are 1 and add their values. For example, 10110101 in binary:

Binary → Decimal Conversion
// Column values (powers of 2, right to left) 128 64 32 16 8 4 2 1 1 0 1 1 0 1 0 1 // Add the columns where bit = 1 128 + 32 + 16 + 4 + 1 = 181 // So 10110101 (binary) = 181 (decimal) = B5 (hex)

Hexadecimal uses the letters A=10, B=11, C=12, D=13, E=14, F=15 for values above 9. One hex digit represents exactly 4 binary bits (a nibble), making hex a compact shorthand for binary — which is why Unity and web colour codes like #FF6347 are written in hex.

🎮
Hex in Unity — Colours & Flags

Unity's Color32 stores RGBA values as bytes (0–255). In the Unity Inspector you'll see colours like #39E8A2FF — each pair of hex digits is one colour channel. 39 = red (57), E8 = green (232), A2 = blue (162), FF = alpha (255 = fully opaque). Bit flags in game state bitmasks also use hex for compactness.

Section 2.2

Two's Complement

Plain binary can only represent positive integers. To represent negative numbers, computers use two's complement — the standard method for signed integers in virtually all modern processors.

Two's Complement Method — 3 Steps
To find the two's complement representation of a negative number −N:

Step 1: Write the positive value N in binary (8 bits).
Step 2: Flip all the bits (0→1, 1→0). This is called the one's complement.
Step 3: Add 1 to the result.

The most significant bit (MSB) — the leftmost bit — acts as the sign bit. If it is 1, the number is negative. If 0, it is positive.

With 8-bit two's complement, you can represent integers from −128 to +127. The range is asymmetric because zero takes up one of the positive slots.

Activity 2.2 — Two's Complement Calculator Step-by-step
Enter a negative integer (−128 to −1) to see the two's complement method step by step.
Range: −128 to −1

To decode a two's complement number: if the MSB is 1, apply the same process in reverse (flip bits, add 1) to find the magnitude, then apply a negative sign. If the MSB is 0, read it as a normal positive binary value.

HSC Exam Tip
Exam questions typically ask you to represent a negative integer in 8-bit two's complement, or to determine the decimal value of a given two's complement binary string. Always show your working — Step 1 (positive binary), Step 2 (flip bits), Step 3 (add 1) — and identify the sign bit.
Section 2.3

ASCII & Unicode

Computers store text as numbers. ASCII (American Standard Code for Information Interchange) assigns a number to each character. Standard ASCII uses 7 bits (128 characters), covering uppercase A–Z (65–90), lowercase a–z (97–122), digits 0–9 (48–57), and control characters.

Key ASCII Ranges to Know
Uppercase A–Z: 65–90  ·  ord('A') = 65
Lowercase a–z: 97–122  ·  ord('a') = 97
Digits 0–9: 48–57  ·  ord('0') = 48
Space: 32  ·  Null: 0

Notice: lowercase letters are exactly 32 higher than uppercase — you can change case by adding or subtracting 32 from the ASCII value, or by flipping bit 5.

Unicode extends ASCII to cover every writing system in the world — over 140,000 characters. The most common encoding is UTF-8, which uses 1–4 bytes per character and is backwards-compatible with ASCII (the first 128 characters are identical). Python 3 strings are Unicode by default.

🎮
ASCII in Game Development

Simple encryption, text validation, and score encoding in games often exploit ASCII arithmetic. A classic game trick: encode player initials for a high-score table using ASCII values. In Unity, System.Text.Encoding.UTF8 is used whenever game data is serialised to/from text files or network packets.

Python — ASCII & Unicode
# Convert character to ASCII code print(ord('A')) # → 65 print(ord('a')) # → 97 (32 higher than 'A') # Convert ASCII code to character print(chr(65)) # → 'A' print(chr(65 + 32)) # → 'a' (to lowercase) # Uppercase without .upper() — flip bit 5 letter = 'g' upper = chr(ord(letter) - 32) print(upper) # → 'G' # Unicode — Python 3 strings are always Unicode print(len('café')) # → 4 characters
Section 2.4

Standard Data Types

A data type tells the computer how to interpret a value stored in memory — how many bytes it needs, what operations are valid, and what range of values it can hold. Choosing the correct data type is a core engineering decision that affects memory use, performance, and correctness.

Data Type
Description & range
Game example
Integer
Whole numbers, no decimal point. 32-bit int: −2,147,483,648 to +2,147,483,647.
playerHealth, score, lives, ammoCount
Real / Float
Numbers with a decimal point. Single precision (32-bit float) gives ~7 decimal digits of precision.
positionX, speed, gravity, jumpHeight
Boolean
True or False only. Stored as 1 bit logically (though typically 1 byte in memory).
isAlive, hasKey, gameRunning, isPaused
Char
A single character. In Python, a string of length 1. In C#, a 16-bit Unicode character.
playerInitial, direction ('N','S','E','W')
String
A sequence of characters. Variable length. Stored as an array of char values.
playerName, itemDescription, dialogueText
Date & Time
Always stored as 32 bits (4 bytes). Displayed in formats like DD/MM/YYYY or YYYY-MM-DD hh:mm:ss.
saveTimestamp, achievementUnlocked, sessionStart
Single Precision Floating Point
The syllabus specifically names single precision floating point (32-bit float). It stores numbers in IEEE 754 format: 1 sign bit, 8 exponent bits, 23 mantissa bits. This gives a range of approximately ±3.4 × 10³⁸ but only ~7 significant decimal digits. Unity's Vector3, transform.position, and physics values all use float — which is why precision errors can accumulate in very large game worlds.

Section 2.6

Data Structures

A data structure is a way of organising multiple data items in memory. Choosing the right structure for a problem is one of the most important decisions in software engineering — the wrong choice can make a program slow, fragile, or difficult to maintain.

Data Structures — Visual Explorer Interactive
Array (1D)
Array (2D)
Record
Tree
Sequential File
A 1D array stores items of the same type in a fixed sequence, accessed by an index starting at 0. Ideal for leaderboard scores, inventory lists, or storing a level's tile data.
leaderboard[0..4] — Integer
Python
# Declare and use a 1D array (Python list) leaderboard = [9800, 7400, 6100, 5500, 3200] # Access by index print(leaderboard[0]) # → 9800 (first place) print(leaderboard[4]) # → 3200 (fifth place) # Update a score leaderboard[2] = 6300 # Length print(len(leaderboard)) # → 5
A 2D array is an array of arrays — like a grid or table. Perfect for game maps, chess boards, Sudoku grids, or tile-based level data. Access uses two indices: [row][col].
gameMap[3][5] — Integer (0=empty, 1=wall, 2=pickup)
Python
# 2D array: 3 rows × 5 columns gameMap = [ [1, 1, 1, 1, 1], # row 0 — all walls [1, 0, 0, 2, 1], # row 1 — open space + pickup [1, 1, 1, 1, 1], # row 2 — all walls ] print(gameMap[1][3]) # → 2 (pickup at row 1, col 3) # Traverse the whole grid for row in gameMap: for cell in row: print(cell, end=' ') print()
A record groups related fields of different types under one name. Unlike an array (same type), a record can mix strings, integers, booleans, and floats. Each game entity — a player, enemy, or item — is best represented as a record.
RECORD — Player
playerName"ShadowAce"String

Python — Record as Dictionary
# Python uses a dict to represent a record player = { "playerName" : "ShadowAce", "score" : 14750, "playerHealth": 75, "positionX" : -14.35, "isAlive" : True } print(player["playerName"]) # → "ShadowAce" player["score"] += 500 # update field
A tree is a hierarchical structure where each node has a value and links to child nodes. The topmost node is the root. Trees model hierarchies naturally — a game scene graph (Unity's object hierarchy), a file system, or a decision tree for enemy AI.
Tree Terminology
Root: topmost node (no parent).   Parent / Child: nodes connected by an edge.   Leaf: node with no children.   Depth: number of edges from root to a node.
A sequential file stores records one after another, in a fixed order. To find a specific record you must read from the start — there is no random access. Used for log files, save game data written sequentially, and audit trails.
scores.dat — Sequential File
Record 0 →"ShadowAce"98002026/04/15
Record 1 →"NovaStar"74002026/04/13
Record 2 →"IronByte"61002026/04/10
Sequential vs Random Access
Sequential files must be read in order from the beginning — you cannot jump to record 500 without reading records 0–499 first. For fast lookups, use arrays (index-based random access) or hash tables. Sequential files are simple, portable, and appropriate when all records must be processed anyway (e.g., batch processing end-of-day scores).

Chapter 2 Quiz

Six questions covering number systems, data types, and data structures.

Knowledge Check — Chapter 2 6 Questions
1. What is the decimal value of the binary number 10110100?
A 174
B 180
C 182
D 164
Correct — B: 180. Column values where bits are 1: 128 + 32 + 16 + 4 = 180. Bit pattern: 1×128 + 0×64 + 1×32 + 1×16 + 0×8 + 1×4 + 0×2 + 0×1 = 180.
2. What is the hexadecimal equivalent of decimal 255?
A EF
B FE
C FF
D F0
Correct — C: FF. 255 = 15×16 + 15 = F×16 + F. Each hex digit F represents the maximum nibble value (1111 in binary). 255 = 11111111 in binary — all 8 bits set.
3. Using 8-bit two's complement, what is the binary representation of −12?
A 10001100
B 11110011
C 11110100
D 10000011
Correct — C: 11110100. Step 1: +12 = 00001100. Step 2: Flip bits → 11110011. Step 3: Add 1 → 11110100. The MSB (1) confirms this is a negative number.
4. A game stores a player's position as positionX = 3.75. Which data type is most appropriate?
A Integer
B Boolean
C Single precision floating point
D String
Correct — C: Single precision floating point. Position values require decimal precision — integer cannot store 3.75. Float (real) is appropriate. The syllabus specifically names "single precision floating point" as a required data type.
5. A game's map is stored in a 10×10 grid where each cell contains 0 (empty) or 1 (wall). Which data structure is most appropriate?
A Sequential file
B Two-dimensional array
C Record
D Tree
Correct — B: Two-dimensional array. A 2D array maps naturally to a grid — each cell accessed by [row][column]. All values are the same type (integer), and random access by position is required. Trees model hierarchies; records group mixed-type fields; sequential files lack random access.
6. In the HSC data dictionary format, which column specifies how many characters are needed to display a value on screen?
A Size in bytes
B Format for display
C Size for display
D Validation
Correct — C: Size for display. "Size in bytes" is the storage size in memory. "Size for display" specifies how many characters are needed on screen (e.g., a score with max 9,999,999 needs 7 display characters). These are separate columns in the HSC format.
