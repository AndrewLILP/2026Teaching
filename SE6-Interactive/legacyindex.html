Chapter 01 · Year 11

Thinking Like a Software Engineer

Before writing a single line of code, great software engineers learn to think systematically. In this chapter we explore the mindset, tools, and processes that separate great developers from the rest.

What is Software Engineering?

Software engineering is more than just programming. It is the disciplined application of engineering principles to the design, development, testing, and maintenance of software. Think of it as the bridge between a raw idea and a working, reliable system.

In this course you will learn to think and work like a professional software engineer — planning carefully, writing clean code, testing thoroughly, and reflecting on your process. The game industry is one of the most demanding environments for software engineers, which makes it the perfect lens for this course.

Key Concept
Software engineering applies systematic, disciplined, quantifiable approaches to the development, operation, and maintenance of software. (IEEE Definition)

Software Development Approaches

There is no single way to build software. Different projects require different approaches. The NSW syllabus requires you to understand and apply these key methodologies:

Waterfall Agile RAD Scrum Extreme Programming DevOps
Game Industry Link
Most professional game studios use Agile or Scrum. Work is broken into short cycles called sprints, with a playable build delivered at the end of each cycle. You will use this same approach for your Year 12 Software Engineering Project.

Pseudocode & Flowcharts

Before writing code, engineers communicate their logic using two key tools: pseudocode (structured plain English) and flowcharts (visual diagrams). Both are assessed in the HSC examination.

Here is a pseudocode example of a simple game loop — the core cycle that runs every game:

Pseudocode — Game Loop
// A basic game loop in pseudocode BEGIN GameLoop InitialiseGame() WHILE gameRunning = TRUE ProcessInput() UpdateGameState() RenderFrame() IF playerLives = 0 THEN gameRunning ← FALSE ENDIF ENDWHILE DisplayGameOver() END GameLoop
HSC Exam Tip
In pseudocode, all keywords (BEGIN, END, WHILE, IF, THEN) are written in CAPITALS. Structural elements always come in pairs: BEGIN/END, WHILE/ENDWHILE, IF/ENDIF. Indentation shows the structure of control statements.
