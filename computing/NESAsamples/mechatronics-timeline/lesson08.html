Accessibility: POUR

A widely used accessibility framework groups every consideration into four categories — POUR.

P — Perceivable Can people actually see or hear it? Captions, alt text, sufficient colour contrast.
O — Operable Can people interact with it regardless of input method? Keyboard navigation, no content that requires precise, fast clicking.
U — Understandable Is the language plain and the navigation predictable? No jargon without explanation, no surprising behaviour.
R — Robust Does it keep working across different devices and assistive technology, not just on the one screen you tested it on?

Teacher tip

Watch for "we'll add accessibility at the end" — captions, alt text and keyboard navigation are far easier to build in from the start than retrofit in Lesson 10 under time pressure.

Checkpoint

Three quick questions before you go.

1. Which principle is being used when a button is styled to clearly look pressable?
2. Adding captions to a video is mainly an example of which POUR category?
3. Which of these is the clearest example of "feedback" in UI design?
