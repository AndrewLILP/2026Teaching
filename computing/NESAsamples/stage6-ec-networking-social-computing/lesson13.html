The hook

A new router arrives in its box with every setting at a factory default — including an admin username of "admin" and a password of "admin" or "password." In many cases, these defaults are published online in the manufacturer's documentation, available to anyone who knows the router's model. Thousands of home networks have been compromised not by sophisticated attacks but by households that never changed the default password on the day of setup.

The question for today If you can log into a home router's admin panel without knowing the household's password, what can you actually change from there — and why does that matter more than just changing the Wi-Fi name?

Configuration sequence

Order matters when configuring a new home network. Security steps come before any household member connects — not after.

Worked example

Setting up a new home network — in order

  1. Change the admin credentials immediately. Log into the router admin panel at its default IP address (usually 192.168.0.1 or 192.168.1.1), change the admin username and password to something unique before doing anything else. This closes the door the hook described.
  2. Update router firmware. Manufacturers release firmware updates that patch security vulnerabilities. Check for updates before the network is used — this is the last moment it's safe to do so before any device connects.
  3. Set a strong SSID and Wi-Fi password. Replace the default SSID (which often reveals the router model, narrowing an attacker's search) with a custom name, and set a password of at least 12 characters. WPA3 is the current recommended encryption standard; use it if the router supports it.
  4. Create a guest network for IoT and visitor devices. Enable a second SSID with network segmentation enabled — smart home devices and guests connect here, isolated from the main LAN where sensitive devices live.
  5. Name every device. Log into the router admin panel and assign a recognisable hostname to each device's MAC address. "Living-room-TV" is easier to audit than a string of hexadecimal characters.
  6. Verify the firewall is active. Most routers enable their firewall by default — confirm it's on. Consider disabling remote management (admin access from outside the home) unless there is a specific reason to need it.

Teacher tip

Steps 1 and 2 before anyone else connects is the key discipline here — most home security failures come from skipping setup steps to get online faster. Ask students to identify which of their Lesson 12 requirements each step addresses.

Cybersecurity decisions

Case study

The Mendez household — security decisions

Returning to the Mendez household from Lesson 12: two parents working from home (accessing employer systems), two school-age children and an elderly grandparent with a medical alert device.

Firewall: The router's built-in firewall is confirmed active. One parent works for a healthcare organisation and has also installed a software firewall on her work laptop for additional protection, as required by her employer.

VPN: Both parents connect to their employers' systems via VPN when working from home. The router does not route all household traffic through a VPN — only the parents' laptops use one — to avoid impacting download speeds for the children's school activities.

Network segmentation: Three SSIDs are active: a main household network (parents' work laptops), a children's network (tablets, gaming consoles, school laptops), and an IoT/guest network (the medical alert hub, smart thermostat and any visitor devices).

Password policy: The household has agreed to a minimum 14-character Wi-Fi password changed every 12 months, with the IoT/guest network using a different password to the main and children's networks.

Evaluate: The Mendez setup uses three SSIDs rather than the more common two. What specific threat does the separation of the children's network from the main household network address that the IoT/guest network separation alone would not?

Backup planning

A cybersecurity plan isn't complete without accounting for failure. For a home network, three things need a backup plan:

Three things to plan for Internet outage (the ISP goes down or the router fails) — what does the household do? Mobile hotspot, tethering, or simply waiting, depending on urgency. Data backup (what happens if a device on the network is compromised by ransomware or stolen?) — regular automated backups to a separate location, not just a second drive on the same network. Device failure (the smart lock or medical alert device loses connectivity) — what is the fallback for the physical device while the network is restored?

For the Mendez household, the grandparent's medical alert device has a cellular backup that operates independently of the home Wi-Fi — the household specifically chose a device with this feature after recognising that a network outage cannot affect emergency response capability.
