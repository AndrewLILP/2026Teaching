Number Systems and Data Representation

Every digit you'll place in a Sudoku grid, and every decision your solver makes, ultimately has to exist inside a computer as nothing but ones and zeros. Today you'll learn the three number systems programmers actually work with, and how a computer represents a negative number when plain binary alone can't.

Why number systems?

You already know one number system extremely well — decimal, base 10, the one with ten digits (0–9) that you've used your whole life. A computer's circuits, at the hardware level, don't use decimal at all. A transistor is either on or off, electricity is either flowing or it isn't — so computers represent everything, eventually, in binary: a number system with only two digits, 0 and 1.

The question for today If a computer only really understands ones and zeros, how does it ever end up displaying the number 7 you typed into a Sudoku square — and how does it represent a number like −1, when binary on its own has no minus sign?

Binary, decimal and hexadecimal

Tap each card to reveal what it means.

Let's convert one together first

Decimal to binary: divide repeatedly by 2 and read the remainders from bottom to top.

42 ÷ 2 = 21 remainder 0 21 ÷ 2 = 10 remainder 1 10 ÷ 2 = 5 remainder 0 5 ÷ 2 = 2 remainder 1 2 ÷ 2 = 1 remainder 0 1 ÷ 2 = 0 remainder 1 Reading remainders bottom to top: 101010 Check: 32 + 8 + 2 = 42 ✓

Binary to hexadecimal: pad to a multiple of 4 bits, then convert each group of 4 separately.

101010 padded to 8 bits → 00101010 Split into nibbles: 0010 1010 Each nibble to hex: 2 A So 42 in decimal = 101010 in binary = 2A in hexadecimal

Representing negative integers

Plain binary has no minus sign — it's just a string of 0s and 1s. So how does a computer store −5? The standard method is called two's complement: write the positive version of the number in binary, flip every bit, then add 1.

Let's work through −5 together, using 8 bits

Step 1 — write 5 in 8-bit binary: 00000101 Step 2 — flip every bit: 11111010 Step 3 — add 1: 11111011 So −5 in 8-bit two's complement is 11111011 Check: as an unsigned number, 11111011 = 251. A leading 1 signals "negative", and 251 − 256 = −5 ✓

Why this matters for our solver

Two pieces of today's lesson will show up directly in the Sudoku build. First: many solver designs use a sentinel value — often 0 or −1 — to mean "this square is empty." Representing that sentinel correctly depends on the computer being able to store negative integers properly, which is exactly what two's complement does. Second: every character you'll eventually display on screen — including every digit in the grid — is stored using a character-encoding standard like ASCII or Unicode, which assigns every symbol a number. Number systems aren't just an abstract topic; they're the reason any of this works at all underneath your Python code.
