Data Types and Data Structures

A Sudoku grid is just a 9×9 arrangement of digits — but "just" is doing a lot of work in that sentence. Before you write any code, you need to decide exactly what kind of data each square holds, and exactly what structure will hold all 81 of them together in memory.

Researching & Designing

Before you write any code

Every value your program ever touches needs two decisions made about it: what type of data is it (a number? a piece of text? true or false?), and what structure holds it together with all the related values around it (one value on its own? a list of them? a grid of them?). Get either decision wrong early, and you'll be fighting your own data for the rest of the build.

Data dictionaries

A data dictionary is a tool for describing every piece of data a program uses before you write a line of code — its name, its type, what it represents, and how it relates to other data.

Choosing a structure for our grid

A 9×9 grid is a two-dimensional problem, and two-dimensional problems can feel like a lot to take in at once. Before tackling the full grid, it's worth practising the same kind of structural thinking on something smaller: a one-dimensional word jumble.

Warm-up: the word jumble A scrambled word like "GORAMPRMNI" can be represented as a one-dimensional array — a single list of characters, each one accessed by a single index: letters[0] is "G", letters[1] is "O", and so on.

A Sudoku grid is the same basic idea, just one dimension bigger: instead of one index, each cell needs two — a row and a column. In Python, the natural way to represent that is a list of lists: grid[row][col] gives you the value at that exact square, the same way letters[index] gave you a single letter in the word jumble.

Design the grid structure

Extension · A–B

Justify your choice

Compare a list of lists against at least one alternative (e.g. a single flat 1D array with index maths, or a dictionary keyed by (row, column) pairs), and justify which one you'd actually use for the Sudoku solver and why.

Teacher tip

If a 2D array feels genuinely too abstract for a student today, let them stay with the word jumble version a little longer — the grid version will make more sense once Lesson 7 starts looping over rows and columns explicitly.
