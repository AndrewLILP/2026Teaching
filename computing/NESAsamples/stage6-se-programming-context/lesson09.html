Backtracking, Applied to Sudoku

Last lesson's maze had junctions and dead ends. A Sudoku grid has exactly the same shape of problem — squares instead of junctions, digits instead of directions. Today you'll watch backtracking happen step by step on a real grid, including the exact moment it hits a dead end and has to undo a choice.

Researching & Designing

From maze to grid

In the maze, a dead end meant undoing one choice and trying the next option at the same junction. In Sudoku, a "dead end" means an empty square where none of the remaining values work — and "undoing" means erasing the value in the square that was filled just before it, and trying that square's next value instead. Same idea, different costume.

A mini grid, for now

A real Sudoku grid is 9×9 — far too many steps to watch one at a time today. So the widget below uses a smaller 4×4 version instead: four rows, four columns, four 2×2 boxes, and digits 1–4. Every rule still applies exactly the same way — a value can't repeat in its row, column, or box — it's just small enough to step through completely in one sitting. The exact same logic is what Lesson 10 will scale up to the full 9×9 grid in actual Python code.

Researching & Designing

What you watched

Let's name what happened, step by step

The solver reached row 0, column 3 and tried every value from 1 to 4 — all four were blocked. That's a dead end, exactly like the maze's Left turn at Junction 2 that went nowhere. Backtracking responded the same way it did in the maze: it didn't start over from the very beginning. It undid the single most recent choice — the 3 it had placed at row 0, column 2 — and tried that square's next value instead. That one small change was enough to unblock everything that came after it.
