From pseudocode to Python

You've already written the logic for most of this twice — once as plain English in Lesson 5, once as pseudocode in Lessons 6 and 7. Today's job isn't to invent anything new. It's translation: turning lines you already trust into syntax Python actually understands. By the end of today, you'll have three working functions you can call and test for yourself.

Producing & Implementing

Setting up the grid

Open your IDE and start a new file. The first thing every other function will depend on is the grid itself — the list of lists you designed back in Lesson 4.

STEP 1 Represent the grid
// ── NEW IN STEP 1 ──
grid = [
    [5, 3, 0, 0, 7, 0, 0, 0, 0],
    [6, 0, 0, 1, 9, 5, 0, 0, 0],
    [0, 9, 8, 0, 0, 0, 0, 6, 0],
    [8, 0, 0, 0, 6, 0, 0, 0, 3],
    [4, 0, 0, 8, 0, 3, 0, 0, 1],
    [7, 0, 0, 0, 2, 0, 0, 0, 6],
    [0, 6, 0, 0, 0, 0, 2, 8, 0],
    [0, 0, 0, 4, 1, 9, 0, 0, 5],
    [0, 0, 0, 0, 8, 0, 0, 7, 9]
]
What's new: this is exactly the list-of-lists structure you designed in Lesson 4. grid[row][col] gives you one square's value, and 0 marks an empty square — the same sentinel idea from Lesson 3.
Producing & Implementing

print_grid()

Before writing anything clever, it helps to be able to actually see the grid as you work on it.

STEP 2 Add a way to see the grid
// ── NEW IN STEP 2 ──
grid = [ ... ]  # same 9×9 grid as Step 1

def print_grid(grid):
    for row in grid:
        print(row)
What's new: print_grid() is a procedure — it performs an action (displaying the grid) but doesn't hand back a value. That's exactly the distinction you made in Lesson 6.
Producing & Implementing

find_empty()

This is the Lesson 7 pseudocode for finding the next empty square, translated almost line for line.

Reminder — Lesson 7 pseudocode

FOR row = 0 TO 8
    FOR col = 0 TO 8
        IF grid[row][col] == 0 THEN
            RETURN row, col
        ENDIF
    ENDFOR
ENDFOR
RETURN "no empty square found"
STEP 3 Find the next empty square
// ── NEW IN STEP 3 ──
grid = [ ... ]  # same 9×9 grid as Step 1

def print_grid(grid):
    for row in grid:
        print(row)

def find_empty(grid):
    for row in range(9):
        for col in range(9):
            if grid[row][col] == 0:
                return (row, col)
    return None
What's new: FOR row = 0 TO 8 becomes for row in range(9): — range(9) generates 0, 1, 2, … 8, exactly the same eight values. Returning a tuple makes find_empty() a function, since it hands a result back to whoever called it.
Producing & Implementing

is_valid()

The biggest piece today — the pattern you spotted all the way back in Lesson 5, the one that repeats for every square and every value: checking row, column and box together.

STEP 4 Check whether a value is allowed
// ── NEW IN STEP 4 ──
grid = [ ... ]  # same 9×9 grid as Step 1

def print_grid(grid):
    for row in grid:
        print(row)

def find_empty(grid):
    for row in range(9):
        for col in range(9):
            if grid[row][col] == 0:
                return (row, col)
    return None

def is_valid(grid, row, col, value):
    for c in range(9):
        if grid[row][c] == value:
            return False

    for r in range(9):
        if grid[r][col] == value:
            return False

    box_row_start = (row // 3) * 3
    box_col_start = (col // 3) * 3
    for r in range(box_row_start, box_row_start + 3):
        for c in range(box_col_start, box_col_start + 3):
            if grid[r][c] == value:
                return False

    return True
What's new: three checks, one function — row, column, then box, exactly matching the Lesson 6 pseudocode's AND logic. Notice DIV from the pseudocode became // in Python: integer division that rounds down, exactly what box_row_start and box_col_start need.
Producing & Implementing

Testing our code

Add these three lines underneath everything else and run the file:

print(find_empty(grid)) # (0, 2) — the first empty square print(is_valid(grid, 0, 2, 4)) # True — 4 doesn't conflict with anything print(is_valid(grid, 0, 2, 5)) # False — 5 is already in that row

If your output matches the comments, all three functions are working correctly. Notice what you just did: you desk-checked your code by actually running it, the same way Lesson 6 desk-checked pseudocode by hand — except now the computer does the checking for you.

Teacher tip

If a student's output doesn't match, resist jumping in immediately — ask them to compare their code line by line against the walkthrough first. Real debugging tools arrive in Lessons 12–13; for now, careful comparison is the only tool available, and that's deliberate.
