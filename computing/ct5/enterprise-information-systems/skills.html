The project
Unit project · Sprints 0–5
Task Tracker App

You will design, build, and evaluate a browser-based task tracker — an app that lets users add, complete, filter, and delete tasks, with data that persists between sessions. By Sprint 4 you have a fully working JavaScript app. In Sprint 5, a Python script reads your exported data and produces a summary report. Each sprint page contains NSW pseudocode walkthroughs, annotated code steps, tiered tasks, and notebook prompts.

Sprint 2
JavaScript Producing & Implementing
Objects & Local Storage

Store each task as a JavaScript object in an array. Use localStorage to persist the data across page refreshes — so tasks survive closing the browser. Introduces JSON.stringify and JSON.parse.

Filtering & Searching

Filter the task list by status (All / Active / Done) and search by keyword. Introduces array methods and the key principle that the data array is the source of truth — the display is just a view of it.

Delete, Edit & Complete

Sprint 5
Python Producing & Implementing
Python Data Layer

Export your task array from the browser as a .json file, then write a Python script that reads it, analyses the data (counts, filters, status summaries), and writes a report to .csv. Introduces Python's json and csv modules and file I/O.
