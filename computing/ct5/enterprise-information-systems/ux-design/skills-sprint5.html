Python Data Layer

Your JavaScript app is complete. But it lives entirely in the browser — every analysis has to be done manually, one task at a time. In this sprint you add a standalone Python script that acts as a data layer: it reads your exported task data, analyses it (counts, filters, summaries by status), and writes a structured report to a .csv file. This is your first taste of how the two languages work together — each doing what it's best at.

Why Python for this? JavaScript runs in the browser and is excellent at responding to user interaction. Python runs on your computer, is excellent at processing files, and has a rich ecosystem for data analysis. In professional software, different languages in a system each handle what they do best. This sprint introduces that idea at a small scale.
Identifying & Defining

Key concepts

json module

Python's built-in module for reading and writing JSON. json.load(file) reads a JSON file and returns a Python object (list or dict). json.dumps(obj) converts a Python object back to a JSON string.

import json with open('tasks.json') as f: tasks = json.load(f) # tasks is now a Python list of dicts
csv module

Python's built-in module for reading and writing CSV files. csv.writer(file) returns a writer object. writer.writerow(list) writes one row. writer.writerows(list_of_lists) writes many rows at once.

import csv with open('report.csv', 'w', newline='') as f: writer = csv.writer(f) writer.writerow(['id', 'text', 'status'])
File I/O with with

The with open(...) as f: pattern is Python's preferred way to work with files. It automatically closes the file when the indented block ends — even if an error occurs. Never leave files open.

# File is always closed after the block with open('tasks.json', 'r') as f: data = json.load(f) # f is closed here — even on error
List comprehension

A concise way to create a filtered or transformed list from an existing one. Equivalent to a for loop with an optional if condition — but in one readable line.

# All completed tasks done = [t for t in tasks if t['completed']] # All task texts as a list texts = [t['text'] for t in tasks]

Pseudocode first — the full pipeline algorithm

The script has two parts: the JS side adds an export button; the Python side reads, analyses, and writes. Read through both before writing any code.

// Sprint 5 — NSW pseudocode // PART A: JavaScript — add export button to index.html + app.js PROCEDURE exportTasks() jsonString ← CONVERT tasks TO JSON (formatted, indented) blob ← CREATE downloadable object from jsonString with type "application/json" url ← CREATE temporary download URL from blob CREATE invisible <a> link pointing to url, filename = "tasks.json" TRIGGER click on the <a> link RELEASE url ENDPROCEDURE // PART B: Python — analyse.py PROCEDURE main() // 1. NEW IN STEP 1b — exportTasks and export button listener
// ── exportTasks: create a downloadable JSON file from the tasks array ──
function exportTasks() {
  if (tasks.length === 0) {
    alert('No tasks to export.');
    return;
  }
  const jsonString = JSON.stringify(tasks, null, 2);  // 2-space indented
  const blob       = new Blob([jsonString], { type: 'application/json' });
  const url        = URL.createObjectURL(blob);

  // Programmatically click an invisible link to trigger download
  const a  = document.createElement('a');
  a.href     = url;
  a.download = 'tasks.json';
  a.click();
  URL.revokeObjectURL(url);  // release memory
}

// Add to initialise():
document.getElementById('export-btn')
        .addEventListener('click', exportTasks);
JSON.stringify(tasks, null, 2) — the second argument is a replacer (unused here, so null); the third is the indent level. Without it, the output is one long line — hard for Python (and humans) to read. With 2, each property is on its own indented line.

URL.revokeObjectURL(url) — the object URL is a temporary reference that holds a chunk of memory. Calling revokeObjectURL immediately after the click releases that memory. Without this, the URL (and the data it references) stays in memory until the tab closes.

Test checkpoint: Click the export button with some tasks added. A file named tasks.json should download. Open it in a text editor and verify it contains valid, indented JSON matching your tasks.

    # ── 1. Read the exported JSON file ──
    with open('tasks.json', 'r', encoding='utf-8') as f:

    print(f'Loaded {len(tasks)} tasks.')

encoding='utf-8' — always specify encoding when opening text files. Without it, Python uses the system default, which differs between Windows (often cp1252) and macOS/Linux (utf-8). Tasks with emoji or accented characters will cause errors on Windows without this argument.

if __name__ == '__main__': — this guard means main() only runs when the script is executed directly, not when it is imported by another script. It's a Python convention that makes scripts reusable as modules later.

Test checkpoint: Run python analyse.py from the terminal. You should see the task count and first task printed. If you get FileNotFoundError, check that tasks.json is in the same folder as analyse.py.

Step 3 — Analyse the data

Add the analysis block below the file-reading code. Count total tasks, completed tasks, active tasks, and calculate the completion percentage.

Step 3 · analyse.py — complete file
NEW IN STEP 3 — analysis block (lines 9–18)
import json

def main():
    # ── 1. Read ──
    with open('tasks.json', 'r', encoding='utf-8') as f:
        tasks = json.load(f)

    # ── 2. Analyse ──
    total     = len(tasks)
    completed = len([t for t in tasks if t['completed']])
    active    = total - completed
    pct_done  = round((completed / total) * 100, 1) if total > 0 else 0

    print(f'Total tasks:     {total}')
    print(f'Completed:       {completed}  ({pct_done}%)')
    print(f'Active:          {active}')


if __name__ == '__main__':
    main()
List comprehension as a counter: len([t for t in tasks if t['completed']]) builds a new list of only the completed tasks, then counts its length. An alternative is sum(1 for t in tasks if t['completed']) — a generator expression that doesn't build a list at all, slightly more memory-efficient for large datasets.

Guard for division by zero: pct_done = ... if total > 0 else 0 — if the task list is empty, dividing by zero would raise a ZeroDivisionError. The ternary expression handles this cleanly.

f-strings: f'Completed: {completed}' is Python's string interpolation syntax. The f prefix tells Python to evaluate expressions inside {}.

Step 4 — Write the CSV report

Add the CSV writing block. The report has one row per task, plus a summary row at the bottom showing totals.

Step 4 · analyse.py — complete file
NEW IN STEP 4 — csv import + write block (lines 2, 20–36)
import json
import csv
from datetime import datetime

def main():
    # ── 1. Read ──
    with open('tasks.json', 'r', encoding='utf-8') as f:
        tasks = json.load(f)

    # ── 2. Analyse ──
    total     = len(tasks)
    completed = len([t for t in tasks if t['completed']])
    active    = total - completed
    pct_done  = round((completed / total) * 100, 1) if total > 0 else 0
    exported  = datetime.now().strftime('%Y-%m-%d %H:%M')

    # ── 3. Write CSV report ──
    with open('report.csv', 'w', newline='', encoding='utf-8') as f:
        writer = csv.writer(f)

        # Header row
        writer.writerow(['id', 'text', 'status', 'exported_at'])

        # One row per task
        for task in tasks:
            status = 'done' if task['completed'] else 'active'
            writer.writerow([
                task['id'],
                task['text'],
                status,
                exported,
            ])

        # Blank separator then summary row
        writer.writerow([])
        writer.writerow(['SUMMARY', f'Total: {total}',
                          f'Done: {completed} ({pct_done}%)',
                          f'Active: {active}'])

    print(f'Total tasks:  {total}')
    print(f'Completed:    {completed}  ({pct_done}%)')
    print(f'Active:       {active}')
    print(f'Report saved: report.csv')

if __name__ == '__main__':
    main()
newline='' — required by Python's csv module on Windows. Without it, the writer adds an extra blank line between each row. The csv module handles its own newline characters internally, so the file must be opened without Python adding extra ones.

strftime('%Y-%m-%d %H:%M') — formats the current datetime as an ISO-style string like 2026-06-17 14:35. The format codes: %Y = 4-digit year, %m = month, %d = day, %H = 24-hour hour, %M = minutes.

Step 5 — Complete script with error handling

The final version wraps the file-read in a try/except block. A missing or corrupted file should produce a clear message — not a stack trace.

Step 5 · analyse.py — complete final file
NEW IN STEP 5 — try/except error handling
import json
import csv
from datetime import datetime

def main():
    # ── 1. Read JSON ──
    try:
        with open('tasks.json', 'r', encoding='utf-8') as f:
            tasks = json.load(f)
    except FileNotFoundError:
        print('Error: tasks.json not found.')
        print('Export your tasks from the app first.')
        return
    except json.JSONDecodeError as e:
        print(f'Error: tasks.json is not valid JSON. {e}')
        return

    # ── 2. Analyse ──
    total     = len(tasks)
    completed = len([t for t in tasks if t['completed']])
    active    = total - completed
    pct_done  = round((completed / total) * 100, 1) if total > 0 else 0
    exported  = datetime.now().strftime('%Y-%m-%d %H:%M')

    # ── 3. Write CSV ──
    with open('report.csv', 'w', newline='', encoding='utf-8') as f:
        writer = csv.writer(f)
        writer.writerow(['id', 'text', 'status', 'exported_at'])
        for task in tasks:
            status = 'done' if task['completed'] else 'active'
            writer.writerow([task['id'], task['text'], status, exported])
        writer.writerow([])
        writer.writerow(['SUMMARY', f'Total: {total}',
                          f'Done: {completed} ({pct_done}%)',
                          f'Active: {active}'])

    print(f'Total tasks:  {total}')
    print(f'Completed:    {completed}  ({pct_done}%)')
    print(f'Active:       {active}')
    print(f'Report saved: report.csv')

if __name__ == '__main__':
    main()
try/except — catching specific exceptions rather than a bare except: is best practice. It lets expected errors produce clear messages while unexpected errors still propagate normally. Two exceptions are handled here: FileNotFoundError (file doesn't exist) and json.JSONDecodeError (file exists but isn't valid JSON — e.g. it was corrupted or is the wrong file).

Bug detective

Each Python snippet has a deliberate bug. Identify the problem before revealing.

🐛 Bug 1 — missing encoding argument
with open('tasks.json', 'r') as f: tasks = json.load(f)
Works fine on the developer's Mac. Fails on a student's Windows PC with a UnicodeDecodeError. Why?
Bug: Without encoding='utf-8', Python uses the system's default encoding. On macOS and Linux this is UTF-8. On Windows it is often cp1252 (Windows-1252). If any task text contains a character outside cp1252 — an emoji, an accented vowel, a smart quote — the file cannot be decoded and Python raises a UnicodeDecodeError. Fix: always specify encoding='utf-8' explicitly, matching the encoding used when the JS app created the JSON.
🐛 Bug 2 — missing newline='' in csv.writer
with open('report.csv', 'w', encoding='utf-8') as f: writer = csv.writer(f) writer.writerow(['id', 'text', 'status'])
On Windows, the CSV opens in Excel with blank rows between every data row. On macOS it looks fine. What causes this and how do you fix it?
Bug: Python's csv.writer writes its own line endings (\r\n on Windows). When the file is opened in text mode (the default), Python also translates \n to \r\n on Windows. The result is \r\r\n — a carriage return, another carriage return, then a newline — which Excel interprets as two line endings, producing a blank row between every data row. Fix: open the file with newline='' to disable Python's universal newline translation and let the csv module handle it.
🐛 Bug 3 — division by zero
total = len(tasks) completed = len([t for t in tasks if t['completed']]) pct_done = round((completed / total) * 100, 1) # no guard
The script works for every test run during development. When does it crash, and what error does it produce?
Bug: If the task list is empty (the user exported before adding any tasks), total = 0. Dividing by zero raises a ZeroDivisionError. This never happened during development because the developer always had tasks in the app — but a real user clicking Export on an empty app will crash the script. Fix: use the guard pct_done = round(...) if total > 0 else 0.
🐛 Bug 4 — JSON.stringify without indent in JS
// In exportTasks(): const jsonString = JSON.stringify(tasks); // no indent args
The Python script still reads the file correctly. So is this actually a bug? Argue both sides.
It depends on how you define "bug". Python's json.load() parses both compact and indented JSON identically — the file is functionally correct. So this is not a correctness bug. However, it is a maintainability and debugging problem: when a student opens the exported file to verify it before running the Python script, compact JSON is very hard to read. A single error (a missing bracket, an extra comma) is invisible. Indented JSON makes structural errors immediately obvious. Professional tools like curl and browser DevTools always pretty-print JSON for exactly this reason. The verdict: use JSON.stringify(tasks, null, 2) — it costs nothing and aids debugging significantly.
