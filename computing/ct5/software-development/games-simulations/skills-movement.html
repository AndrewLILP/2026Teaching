Key Vocabulary

These terms appear in the code on this page. Read them first — you'll recognise them when they appear.

Word Bank — Moving Things in Unity

MonoBehaviour
The base class every Unity script inherits from. It gives your script access to Unity's event functions like Start() and Update().
Start()
A Unity event function that runs once when the scene loads. Used to set up initial values — runs before the first frame is drawn.
Update()
A Unity event function that runs once every frame. This is where movement, input, and ongoing game logic live. Could run 60, 120, or 30 times per second depending on the device.
Time.deltaTime
The time in seconds since the last frame. Multiplying movement by Time.deltaTime makes the game run at the same speed regardless of frame rate — this is called frame-rate independence.
Transform
A Unity component that stores a GameObject's position, rotation, and scale. Every GameObject has a Transform. In scripts, transform (lowercase) refers to the Transform on the same object as the script.
Vector3
A data structure that holds three float values: x, y, and z. Used to represent positions, directions, and movement in 3D space. Vector3.zero is (0, 0, 0).
float
A decimal number data type in C#. Speed, time, and positions must be floats (not integers) because they need fractional precision. Write 5f to tell C# a number literal is a float.
Input.GetAxis()
A Unity method that reads a named input axis and returns a value between –1 and 1. Input.GetAxis("Horizontal") responds to A/D keys and left/right arrows. "Vertical" responds to W/S.
SerializeField
An attribute written as [SerializeField] above a private variable. It makes a private variable visible in the Unity Inspector without making it public — good coding practice.
Producing & Implementing

How Unity Works — The MonoBehaviour Lifecycle

Unity runs a continuous loop for as long as your game plays. Your C# scripts plug into that loop at specific points called event functions. On this page you'll use two: Start() and Update().

Scene loads Player presses Play in Unity
↓
Start() Runs once — set up initial values here
↓
Update() Runs every frame — movement & input go here
↺ repeats until game stops
Why does this matter? If you put setup code in Update() it runs every frame — multiplying work and causing bugs. If you put movement code in Start() it only fires once and the object stops. Right method, right job.

Frame Rate and Time.deltaTime

Update() is called once per frame. On a fast machine that might be 120 times per second; on a slow one, 30. If you write transform.Rotate(0, 45, 0) in Update(), the cube rotates 45° per frame — which means it rotates four times faster on the fast machine.

Multiplying by Time.deltaTime converts the value from per frame to per second. The cube rotates at the same real-world speed on every device.

❌ Frame-rate dependent (broken)
// Rotates 45° per FRAME — fast on fast machines
transform.Rotate(0, 45f, 0);
✓ Frame-rate independent (correct)
// Rotates 45° per SECOND — consistent everywhere
transform.Rotate(0, 45f * Time.deltaTime, 0);

Data Types — float vs int

Movement values need decimal precision. Speed is not "5 metres per second" — it might be 5.75 at the peak of a jump. Unity's position and rotation methods expect float, not int.

Type Example value Use in this page
float 5f, 0.5f, -3.14f Speed, rotation angle, axis input (–1.0 to 1.0)
int 5, 0, -3 Whole numbers — score, lives, level number
Vector3 new Vector3(0, 1, 0) Direction and position in 3D space

Algorithm 1: Rotating a Cube

Before writing any C#, write the algorithm in NSW pseudocode. This forces you to think about what the code must do before thinking about how to write it.

NSW Pseudocode — RotateCube algorithm
// Rotate a cube around the Y axis every frame
BEGIN RotateCube
  WHILE gameIsRunning DO
    cube.rotation ← cube.rotation + (rotateSpeed × Time.deltaTime)
  ENDWHILE
END RotateCube
Reading the pseudocode The WHILE gameIsRunning loop represents Update() — it repeats every frame. The left-arrow ← is assignment (not equals-comparison). rotateSpeed × Time.deltaTime gives us degrees per second, not degrees per frame.

STEP 3 Add the rotation logic in Update()
✦ NEW IN STEP 3 — one line inside Update() does all the work
using UnityEngine;

public class RotateCube : MonoBehaviour
{
    [SerializeField] private float rotateSpeed = 90f;

    void Start()
    {

    }

    void Update()
    {
        transform.Rotate(0f, rotateSpeed * Time.deltaTime, 0f);   // ── NEW IN STEP 3 ──
    }
}
transform.Rotate(x, y, z) rotates the GameObject by the given number of degrees on each axis this frame. The arguments are (0, y, 0) — only the Y axis rotates (spinning left/right around the vertical). rotateSpeed * Time.deltaTime converts from degrees-per-second to degrees-this-frame. Try changing 0f, rotateSpeed…, 0f to rotateSpeed…, 0f, 0f — the cube will tumble forward instead.
Press Play. Your cube should spin at 90° per second. Select the Cube in the Hierarchy while the game runs and change Rotate Speed in the Inspector — you'll see the change live. This is why [SerializeField] is useful: tweak without stopping and recompiling.

Algorithm 2: Moving a Player

Player movement adds a new idea: reading input. The algorithm reads the player's key presses, converts them into a direction, and then applies movement — but only if the player is actually pressing something.

NSW Pseudocode — MovePlayer algorithm
BEGIN MovePlayer
  WHILE gameIsRunning DO
    horizontal ← INPUT("Horizontal")   // A/D or ←/→ keys: returns –1 to 1
    vertical   ← INPUT("Vertical")     // W/S or ↑/↓ keys: returns –1 to 1
    moveDir    ← Vector3(horizontal, 0, vertical)
    IF moveDir.magnitude > 0.1 THEN
      player.position ← player.position + (moveDir × speed × deltaTime)
    ENDIF
  ENDWHILE
END MovePlayer
Reading the pseudocode — two things to notice The IF moveDir.magnitude > 0.1 guard prevents tiny floating-point drifts from moving the player when no key is pressed (joystick dead zone). The Y component is 0 because we're moving on a flat plane — gravity is handled by Unity's physics, not this script.

    void Start()

    void Update()

}
Input.GetAxis("Horizontal") returns a float from –1 (A key / left arrow) to 1 (D key / right arrow). Zero when no key is pressed. "Horizontal" and "Vertical" are Unity's default axis names — you can see them in Edit → Project Settings → Input Manager. These are the C# equivalent of INPUT("Horizontal") in the pseudocode.
STEP 4 Build the direction Vector3 and apply movement
✦ NEW IN STEP 4 — combine input into a direction and move the player
using UnityEngine;

public class PlayerMovement : MonoBehaviour
{
    [SerializeField] private float speed = 5f;

    void Start()
    {

    }

    void Update()
    {
        float horizontal = Input.GetAxis("Horizontal");
        float vertical   = Input.GetAxis("Vertical");
        Vector3 moveDir = new Vector3(horizontal, 0f, vertical);   // ── NEW IN STEP 4 ──
        transform.Translate(moveDir * speed * Time.deltaTime);      // ── NEW IN STEP 4 ──
    }
}
new Vector3(horizontal, 0f, vertical) puts our input values into the X and Z axes — Y stays zero so the player doesn't float up or sink. transform.Translate() moves the object by the given vector this frame. moveDir * speed * Time.deltaTime scales the direction by how fast we want to move (speed) and how much time has passed since last frame (deltaTime) — exactly as in the pseudocode: moveDir × speed × deltaTime.
STEP 5 Add the dead zone guard — final version
✦ NEW IN STEP 5 — IF guard prevents drift when no key is pressed
using UnityEngine;

public class PlayerMovement : MonoBehaviour
{
    [SerializeField] private float speed = 5f;

    void Start()
    {

    }

    void Update()
    {
        float horizontal = Input.GetAxis("Horizontal");
        float vertical   = Input.GetAxis("Vertical");
        Vector3 moveDir  = new Vector3(horizontal, 0f, vertical);

        if (moveDir.magnitude > 0.1f)                                // ── NEW IN STEP 5 ──
        {                                                            // ── NEW IN STEP 5 ──
            transform.Translate(moveDir * speed * Time.deltaTime);
        }                                                            // ── NEW IN STEP 5 ──
    }
}
moveDir.magnitude is the length of the vector — roughly how fast we're pressing. When no key is held, Input.GetAxis returns near-zero values but not always exactly zero (especially on gamepads). The > 0.1f check creates a dead zone. This matches the IF moveDir.magnitude > 0.1 THEN in the pseudocode. Press Play — WASD or arrow keys should now move the capsule across the scene.

Bug Detective

The script below has three bugs — one of each type. Find them before revealing the answers. Write your findings in your project notebook.

Bug types — quick reference Syntax: code that breaks the rules of C# — won't compile. · Logical: code that compiles and runs but produces wrong behaviour. · Run-time: code that compiles but crashes during Play mode.
🔍 Bug 1 — Syntax Error

RotateCube.cs — missing semicolon

This script will not compile. Find the syntax error on the highlighted line.

using UnityEngine;

public class RotateCube : MonoBehaviour
{
    [SerializeField] private float rotateSpeed = 90f;

    void Start()
    {

    }

    void Update()
    {
        transform.Rotate(0f, rotateSpeed * Time.deltaTime, 0f)   // ← broken line
    }
}
Bug found: Missing semicolon at the end of the transform.Rotate() call. C# statements must end with ;. The fix: transform.Rotate(0f, rotateSpeed * Time.deltaTime, 0f);
🔍 Bug 2 — Logical Error

RotateCube.cs — wrong axis, cube spins into ground

This script compiles and runs, but the cube tumbles forward and disappears into the floor instead of spinning left/right. Which argument is wrong?

using UnityEngine;

public class RotateCube : MonoBehaviour
{
    [SerializeField] private float rotateSpeed = 90f;

    void Start()
    {

    }

    void Update()
    {
        transform.Rotate(rotateSpeed * Time.deltaTime, 0f, 0f);   // ← wrong axis order
    }
}
Bug found: The rotation is applied to the X axis (first argument) instead of the Y axis (second argument). X rotation tilts the object forward/backward; Y rotation spins it left/right. Fix: transform.Rotate(0f, rotateSpeed * Time.deltaTime, 0f);
🔍 Bug 3 — Run-time Error

PlayerMovement.cs — NullReferenceException

This script compiles. When you press Play, Unity crashes with: NullReferenceException: Object reference not set to an instance of an object. The script references a separate Transform variable that was never assigned.

using UnityEngine;

public class PlayerMovement : MonoBehaviour
{
    [SerializeField] private float speed = 5f;
    [SerializeField] private Transform playerTransform;   // ← never assigned in Inspector

    void Start()
    {

    }

    void Update()
    {
        float horizontal = Input.GetAxis("Horizontal");
        float vertical   = Input.GetAxis("Vertical");
        Vector3 moveDir  = new Vector3(horizontal, 0f, vertical);

        if (moveDir.magnitude > 0.1f)
        {
            playerTransform.Translate(moveDir * speed * Time.deltaTime);   // ← crashes here
        }
    }
}
Bug found: playerTransform is declared as a [SerializeField] but was never dragged into the Inspector slot — it's null. Calling .Translate() on a null reference crashes Unity at runtime. Two fixes: (a) drag the Capsule's Transform into the Inspector field before pressing Play, or (b) simplify the script — since the script is on the player, use transform.Translate() (the built-in shortcut to this object's own Transform) and remove the extra variable entirely.
