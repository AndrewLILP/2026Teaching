Identifying & Defining

Getting started with VEX VR

Access: Open vr.vex.com in Chrome or Edge — no install required. Sign in with your school Google account. Your code is saved to your account automatically.
  • Playground / World selector (top left): Choose your environment. For Castle Crasher, your teacher will specify the world — typically the Maze World or Dynamic Environment.
  • Code panel (left): Write your program here. Toggle between Blocks and Python using the button at the top of the code panel.
  • Device panel: Click the robot icon to see your virtual devices. This is where you check port assignments — if your code references Port 1 but the sensor is on Port 3, nothing works.
  • Run / Stop buttons: Green play button starts your program. Red stop button ends it. The robot resets to its starting position each time you run.
  • Brain screen (in simulation): Use brain.screen.print() to display sensor values as your program runs. Drag the brain display open in the simulation panel.
  • Console: In Python mode, print() statements appear in the console below the code — useful for debugging variable values without stopping the program.
Persistence: VEX VR saves your code per world per account. If you switch worlds, your code for the previous world is still there when you switch back. Always check you're in the right world before running.

VEX VR provides a virtual robot with pre-configured devices. The default VR robot typically includes:

  • Left Motor (Port 8): Left drive wheel
  • Right Motor (Port 3): Right drive wheel — note the True reversal flag in Python
  • Gyro (Port 7): For precise angle-based turning
  • Distance Sensor (front, Port 1): Front-facing ultrasonic — returns mm
  • Eye / Colour Sensor (downward): Detects colour of the surface below the robot
SmartDrive vs individual motors: VEX VR supports a SmartDrive object that combines both motors and the gyro into a single drivetrain with methods like drivetrain.drive() and drivetrain.turn(). For Castle Crasher, you'll use SmartDrive. Direct motor control (left_motor.spin()) is available for advanced tasks requiring independent wheel speed.
Python setup Standard VEX VR device initialisation 
from vex import *

# ── Devices ────────────────────────────────────────────────────
brain           = Brain()
left_motor      = Motor(Ports.PORT8)
right_motor     = Motor(Ports.PORT3, True)     # True = reversed
gyro            = Gyro(Ports.PORT7)
distance_sensor = Distance(Ports.PORT1)
eye_sensor      = Eye(Ports.PORT6)

# ── SmartDrive (combines motors + gyro) ────────────────────────
drivetrain = SmartDrive(left_motor, right_motor, gyro)
Copy this block into every new VEX VR Python program as your starting template. Port numbers must match your device panel configuration. The True flag on the right motor tells VEX that this motor is physically mounted in reverse — without it, the motors fight each other when you try to drive forward.

VEX API quick reference

These are the commands you'll use most in VEX VR. Keep this open while coding — refer to it before reaching for Google.

Drivetrain
drivetrain.drive(FORWARD) Start driving
drivetrain.drive(REVERSE) Reverse
drivetrain.stop() Stop motors
drivetrain.set_drive_velocity(50, PERCENT) Set speed %
drivetrain.turn(RIGHT) Start turning
drivetrain.turn_for(RIGHT, 90, DEGREES) Turn exact angle
drivetrain.drive_for(FORWARD, 200, MM) Drive exact distance
drivetrain.set_turn_velocity(30, PERCENT) Set turn speed %
Distance sensor
distance_sensor.get_distance(MM) Read mm → int
distance_sensor.get_distance(CM) Read cm → float
distance_sensor.object_distance(MM) Alt. read method
distance_sensor.is_object_detected() True / False
Eye / Colour sensor
eye_sensor.get_color() Returns Color object
eye_sensor.get_brightness() 0–100 brightness
Color.RED Colour constants
Color.BLUE —
Color.GREEN —
Wait & timing
wait(2, SECONDS) Pause 2 s
wait(500, MSEC) Pause 500 ms
wait(20, MSEC) Loop delay (always!)
Brain / Debug
brain.screen.print("text") Display on brain
brain.screen.clear_screen() Clear display
brain.screen.set_cursor(1, 1) Move cursor
print(variable) Console output
Gyro
gyro.heading(DEGREES) Current heading 0–359°
gyro.rotation(DEGREES) Total rotation
gyro.calibrate() Reset / calibrate
drive() is non-blocking — drive_for() is blocking: drivetrain.drive(FORWARD) starts the motors and the next line runs immediately. drivetrain.drive_for(FORWARD, 200, MM) waits until the robot has moved 200mm before continuing. Use drive_for() for scripted distances; use drive() + a separate wait() when you want to do something else while moving.

📋 Challenge brief

Castle Crasher

The castle is under attack — but the gates are sealed and the paths are blocked. Your autonomous robot must navigate through the maze, avoid the walls, and reach the far end of the castle courtyard without any human input once the program starts.

You do not know the exact layout of the maze in advance. Your robot must respond to what it detects in real time using sensor data — a scripted sequence of movements will not reliably complete the course.

Functional requirements

  • Detect walls/obstacles at ≤ 30cm
  • Stop before any collision
  • Turn to find a clear path
  • Continue driving after avoiding
  • Complete the maze without human input

Non-functional requirements

  • Complete in under 3 minutes
  • No wall contacts during run
  • Program uses at least one function
  • NSW pseudocode in project notebook before implementation
  • Sprint log completed after testing
VEX VR world: Your teacher will specify which VEX VR environment to use. The Maze World is recommended for Castle Crasher — it provides a randomised enclosed maze with walls the distance sensor can reliably detect. Open vr.vex.com, select your world, and verify your device configuration before writing any code.

🔍 Analysis activity — read before you code

Annotate this algorithm

Before writing your own solution, read through the starter algorithm below. For each numbered point in the walkthrough, answer the questions in your project notebook.

  1. What happens if the robot detects a wall on the left AND the front simultaneously? Does this algorithm handle it?
  2. The robot only turns right when it hits a wall. What type of maze would this fail on? (Hint: think about dead ends.)
  3. What value should STOP_DISTANCE be set to, and why? What happens if it's too small? Too large?
  4. The algorithm uses WHILE True — how would you ever stop the robot when it reaches the goal?
  5. Propose one improvement to the algorithm and write the updated pseudocode.
NSW Pseudocode — Castle Crasher starter algorithm
CONSTANT STOP_DISTANCE ← 30      ← threshold in cm
CONSTANT DRIVE_SPEED   ← 50      ← motor speed %
CONSTANT TURN_DURATION ← 0.6     ← seconds

FUNCTION avoidObstacle()
  stopMotors()
  wait(200ms)
  turnRight(speed: DRIVE_SPEED, duration: TURN_DURATION)
  wait(TURN_DURATION seconds)
ENDFUNCTION

FUNCTION getDistanceCM()
  RETURN distanceSensor.read() / 10
ENDFUNCTION

BEGIN
  WHILE True DO
    distance ← getDistanceCM()

    IF distance < STOP_DISTANCE THEN
      avoidObstacle()
    ELSE
      driveForward(speed: DRIVE_SPEED)
    ENDIF

    wait(20ms)
  ENDWHILE
END

Trace & test

Step 10 is the key question: After turning right, what is the distance sensor reading? If the robot has turned to face another wall, the algorithm immediately calls avoidObstacle() again — it will keep spinning until it finds a clear path. Is that the behaviour you want? How would you modify the algorithm to handle this more elegantly?
Producing & Implementing

Code walkthrough — building Castle Crasher

The walkthrough below builds the Castle Crasher program in four steps. Each step adds new lines — green highlighting shows what's new. Type this out yourself rather than copying — the process of typing helps you notice things copy-paste hides.

Building in Blocks: Each step below describes which blocks to add. Blocks auto-snap together — if a block won't connect, check you're dropping it onto the correct slot type (statement slot vs value slot vs boolean slot).
Step 01 Setup — variables and forever loop
── NEW IN STEP 1 — all blocks below are new ──
WHEN STARTED:
  [set DRIVE_SPEED to 50]        ← Variables category
  [set STOP_DISTANCE to 30]      ← Variables category

WHEN STARTED:
  [forever]                      ← Control category
    [set distance to [Distance Sensor value (cm)]]
    ← Sensing category — drag into the value slot
Use two separate "when started" hat blocks — one for variable setup, one for the forever loop. Create DRIVE_SPEED and STOP_DISTANCE as variables in the Variables category first, then drag "set variable to" blocks into your setup hat.
Step 02 Add IF/ELSE decision
── NEW IN STEP 2 ──
  [forever]
    [set distance to [Distance Sensor value (cm)]]
    [if [distance < 30] then]    ← Control: if/else; Operator: <
      [stop driving]             ← Motion category
    [else]
      [drive FORWARD at DRIVE_SPEED %]
    [end if]
    [wait 20 milliseconds]       ← Control category
Drag the "if/else" block inside the forever block. For the condition, use an Operator "<" block — drag the distance variable into the left slot and type 30 into the right. Test this now before adding the avoidObstacle function.
Step 03 Define avoidObstacle function
── NEW IN STEP 3 ──

  [wait 200 milliseconds]
  [turn RIGHT for 0.6 seconds at DRIVE_SPEED %]

  [forever]

      [avoidObstacle]            ← replace [stop driving] with this
    [else]
      [drive FORWARD at DRIVE_SPEED %]
    [end if]
    [wait 20 milliseconds]
Go to My Blocks → Make a Block → name it avoidObstacle (no inputs needed). The define block appears in the editor — build the avoidance sequence inside it. Then replace the bare stop driving in your loop with the new avoidObstacle caller block. Test again.
Step 04 Add brain screen debugging
── NEW IN STEP 4 ──
  [forever]
    [set distance to [Distance Sensor value (cm)]]
    [print [join "dist: " [distance]] ▼ on brain screen]
    [if [distance < 30] then]
      [avoidObstacle]
    [else]
      [drive FORWARD at DRIVE_SPEED %]
    [end if]
    [wait 20 milliseconds]
Adding a brain screen print lets you watch the distance sensor reading in real time as the robot moves. Use Looks → "print on brain screen", and combine text using an Operator "join" block. Now you can see whether the sensor is reading what you expect — essential for tuning your STOP_DISTANCE value.
Step 01 Imports, devices, and constants
── NEW IN STEP 1 — full file shown ──
from vex import *

# ── Devices ────────────────────────────────────────────────────
brain           = Brain()
left_motor      = Motor(Ports.PORT8)
right_motor     = Motor(Ports.PORT3, True)
gyro            = Gyro(Ports.PORT7)
distance_sensor = Distance(Ports.PORT1)
drivetrain      = SmartDrive(left_motor, right_motor, gyro)

# ── Constants ─────────────────────────────────────────────────
DRIVE_SPEED    = 50     # motor speed percent
STOP_DISTANCE  = 30     # cm — stop if closer than this
TURN_DURATION  = 0.6    # seconds per avoidance turn
LOOP_DELAY     = 20     # ms — main loop delay
Constants in UPPER_CASE: STOP_DISTANCE = 30 makes your intent clear and means you can tune the value in one place. Never scatter raw numbers like 30 throughout your program — a change needs to happen in every location, and you'll miss one.
Step 02 Helper function — getDistanceCM
── NEW IN STEP 2 ──
from vex import *

brain           = Brain()
left_motor      = Motor(Ports.PORT8)
right_motor     = Motor(Ports.PORT3, True)
gyro            = Gyro(Ports.PORT7)
distance_sensor = Distance(Ports.PORT1)
drivetrain      = SmartDrive(left_motor, right_motor, gyro)

DRIVE_SPEED    = 50
STOP_DISTANCE  = 30
TURN_DURATION  = 0.6
LOOP_DELAY     = 20

def get_distance_cm():
    """Return front distance sensor reading in centimetres."""
    return distance_sensor.get_distance(MM) / 10
The VEX distance sensor returns millimetres. Dividing by 10 converts to centimetres, which matches our pseudocode and makes threshold values intuitive (30 cm, not 300 mm). Wrapping this in a function means you never have to remember the conversion — just call get_distance_cm().
Step 03 avoidObstacle function
── NEW IN STEP 3 ──
from vex import *

brain           = Brain()
left_motor      = Motor(Ports.PORT8)
right_motor     = Motor(Ports.PORT3, True)
gyro            = Gyro(Ports.PORT7)
distance_sensor = Distance(Ports.PORT1)
drivetrain      = SmartDrive(left_motor, right_motor, gyro)

DRIVE_SPEED    = 50
STOP_DISTANCE  = 30
TURN_DURATION  = 0.6
LOOP_DELAY     = 20

def get_distance_cm():
    """Return front distance sensor reading in centimetres."""
    return distance_sensor.get_distance(MM) / 10

def avoid_obstacle():
    """Stop and turn right to clear a detected obstacle."""
    drivetrain.stop()
    wait(200, MSEC)
    drivetrain.set_turn_velocity(DRIVE_SPEED, PERCENT)
    drivetrain.turn(RIGHT)
    wait(TURN_DURATION, SECONDS)
    drivetrain.stop()
Stop before turning: Without the initial drivetrain.stop(), the robot will try to turn while still moving forward — it will curve rather than turn in place. The wait(200, MSEC) gives the drivetrain time to fully decelerate before the turn begins.
Step 04 Main loop — the complete program
── NEW IN STEP 4 ──
from vex import *

brain           = Brain()
left_motor      = Motor(Ports.PORT8)
right_motor     = Motor(Ports.PORT3, True)
gyro            = Gyro(Ports.PORT7)
distance_sensor = Distance(Ports.PORT1)
drivetrain      = SmartDrive(left_motor, right_motor, gyro)

DRIVE_SPEED    = 50
STOP_DISTANCE  = 30
TURN_DURATION  = 0.6
LOOP_DELAY     = 20

def get_distance_cm():
    """Return front distance sensor reading in centimetres."""
    return distance_sensor.get_distance(MM) / 10

def avoid_obstacle():
    """Stop and turn right to clear a detected obstacle."""
    drivetrain.stop()
    wait(200, MSEC)
    drivetrain.set_turn_velocity(DRIVE_SPEED, PERCENT)
    drivetrain.turn(RIGHT)
    wait(TURN_DURATION, SECONDS)
    drivetrain.stop()

# ── Main control loop ─────────────────────────────────────────
drivetrain.set_drive_velocity(DRIVE_SPEED, PERCENT)

while True:
    distance = get_distance_cm()

    # Debug — watch distance on brain screen and console
    brain.screen.set_cursor(1, 1)
    brain.screen.print("dist:", distance, "cm   ")
    print("distance:", distance)

    if distance < STOP_DISTANCE:
        avoid_obstacle()
    else:
        drivetrain.drive(FORWARD)

    wait(LOOP_DELAY, MSEC)
Set velocity once before the loop: drivetrain.set_drive_velocity(DRIVE_SPEED, PERCENT) sets the speed for all future drive() calls — you don't need to set it inside the loop unless you're changing speed. The debug prints let you watch sensor readings without stopping the program. Remove or comment them out once the robot is working correctly.
