Lap Timing, Checkpoints & Opponent AI

Sprint 1 gave you a kart that handles well — but a track without checkpoints can be cut, and a race without an opponent isn't really a race. This sprint adds ordered checkpoints that can't be skipped, an honest lap timer, and a NavMesh-driven opponent that follows the track without you hand-scripting every turn. By the end, this stops being a driving demo and starts being a race.

Producing & Implementing

Sprint Steps

Work through these in order.

1 Ordered Checkpoints Produce

Place trigger volumes around your track in order, each with a checkpointIndex field (0, 1, 2...). A shared LapManager tracks which index the player is expected to hit next — a checkpoint only counts if it matches, which is what stops players cutting the track or driving backward through an earlier checkpoint for an easy "pass."

Pseudocode 
BEGIN OnTriggerEnter(other)
  IF other.tag = "Player" THEN
    IF checkpointIndex = LapManager.nextExpectedIndex THEN
      LapManager.AdvanceCheckpoint()
    END IF
  END IF
END
C# 
private void OnTriggerEnter(Collider other)
{
    if (other.CompareTag("Player"))
    {
        if (checkpointIndex == LapManager.Instance.nextExpectedIndex)
        {
            LapManager.Instance.AdvanceCheckpoint();
        }
    }
}
This is the same guard pattern, again

Lessons 1–2's collectible checked a tag before acting. Sprint 1's traction checked a speed before turning. This checks an index before advancing. Three different problems, one repeated idea: don't let something happen until the right condition is confirmed. Recognising that pattern is more valuable than memorising any one version of it.

2 Lap Timer Produce

In LapManager, record Time.time when the lap starts. When the final checkpoint completes a lap, calculate the elapsed time, store it in a list of lap times, and reset for the next lap.

Pseudocode 
BEGIN CompleteLap()
  lapTime = Time.time - lapStartTime
  lapTimes.Add(lapTime)
  lapStartTime = Time.time
  nextExpectedIndex = 0
END
C# 
public void CompleteLap()
{
    float lapTime = Time.time - lapStartTime;
    lapTimes.Add(lapTime);
    lapStartTime = Time.time;
    nextExpectedIndex = 0;
}

lapTimes is a List<float>, not a single number — you need every lap recorded, not just the most recent one, for Sprint 3's results screen.

3 NavMesh Opponent AI Produce

Bake a NavMesh over your track surface (Window → AI → Navigation), add a NavMeshAgent to an opponent kart, and give it an ordered array of waypoint Transforms to cycle through — reaching one sends it toward the next, looping back to the start.

Pseudocode 
BEGIN Update()
  IF agent.remainingDistance < arrivalThreshold THEN
    currentWaypoint = (currentWaypoint + 1) mod waypoints.length
    agent.SetDestination(waypoints[currentWaypoint])
  END IF
END
C# 
private void Update()
{
    if (agent.remainingDistance < arrivalThreshold)
    {
        currentWaypoint = (currentWaypoint + 1) % waypoints.Length;
        agent.SetDestination(waypoints[currentWaypoint].position);
    }
}

The % (modulo) operator wraps the index back to 0 once it passes the last waypoint — without it, the array index would eventually go out of range and the AI would crash rather than loop the track forever.

4 Live HUD Produce

Add TextMeshPro elements showing the current lap time (counting up live) and which checkpoint the player has reached (e.g. "Checkpoint 3/6"). Wire both to update from LapManager every frame — a race the player can't see themselves progressing through isn't communicating anything useful.

Testing & Evaluating

Test the Race

The negative tests below specifically try to cheat your own checkpoint system. If you can't cheat it, it's working.

Test CaseStepsExpected ResultActual Result
Full lap, correct order Drive through every checkpoint in order Lap completes, lap time recorded, next lap starts from checkpoint 0 Record here
Skip a checkpoint (negative test) Deliberately drive around one checkpoint, skipping it, then hit the next one The skipped checkpoint is NOT counted — nextExpectedIndex does not advance until the correct checkpoint is hit Record here
Reverse through a checkpoint (negative test) Drive backward through an already-passed checkpoint No progress is granted — nextExpectedIndex is unaffected Record here
Lap timer accuracy Compare a recorded lap time against a manual stopwatch for the same lap Recorded time matches the manual timing within a fraction of a second Record here
Opponent AI full loop Watch the opponent kart drive at least two full laps unassisted It reaches every waypoint in order and loops back to the first without getting stuck or stopping Record here
Sprint Checkpoint

Before you move to Sprint 3, all of the following must be true:

  • All Sprint 1 behaviours (acceleration, steering, drift, wall collision) still work.
  • Checkpoints only advance nextExpectedIndex in the correct order — skipping or reversing grants no progress.
  • Lap times record accurately across at least two consecutive laps.
  • An opponent kart uses NavMeshAgent to loop the full track without getting stuck.
  • The HUD shows live lap time and checkpoint progress.
  • All five test cases above have recorded actual results.

Sprint 2 — Notebook Entry

Sprint 2 crosses two D&P phases — Producing & Implementing and Testing & Evaluating. Your notebook entry needs evidence of both.

Your sprint notebook entry must include:

  1. Sprint log: For each of the four steps, one sentence — what you implemented and whether it worked first time.
  2. Test table: The completed test table from the Testing section, with your actual results recorded.
  3. Bug report: Did you manage to cheat your own checkpoint system on the first attempt? If so, what did you fix? If not, what specifically stopped you?
  4. Design reflection: Race against your opponent AI for a full lap. Does it feel like a genuine competitor, or is it too easy to beat, too hard to beat, or oddly predictable? What's one change you'd make to its waypoints or speed?

Your completed test table must be in your project notebook (physical or digital) before you move to Sprint 3. An untested checkpoint system that can be cheated doesn't go to production.
