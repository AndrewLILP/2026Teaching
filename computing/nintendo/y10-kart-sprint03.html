Identifying & Defining

Race States

Your game now needs exactly one thing to be true at any moment: is the race waiting to start, currently underway, or over? An enum makes this explicit, the same way GameState did for Win/Lose in a platformer — a race just has a different shape of ending.

PreRace
Start screen shown, player input disabled
→ player presses Start
Racing
Movement, checkpoints, lap timer, and opponent AI all active
→ required laps completed
Finished
Results panel shown, input disabled

Notice the race can only ever move forward through these three states, in order — never backward, never skipped. That's a design decision worth stating explicitly in your notebook.

Sprint Steps

Work through these in order.

1 A Second Track Produce

Build a second track layout with a different shape or difficulty — tighter corners, a longer straight, more checkpoints. Reuse your Sprint 1 kart Prefab, Sprint 1 wall Physics Material, and Sprint 2 checkpoint and opponent scripts entirely unchanged — a new track should mean a new arrangement, not new scripts.

  • Reposition your opponent's waypoints to match the new layout — the AI script itself needs no changes, only new waypoint Transforms.
2 Race-Finish State Machine Produce

Build a RaceManager with a RaceState enum. Guard the transition into Finished exactly the way Sprint 2's checkpoints guarded progress — without it, a player crossing the finish line at speed could trigger it more than once before the panel even shows.

Pseudocode 
BEGIN FinishRace()
  IF raceState = Racing THEN
    raceState = Finished
    ShowResultsPanel()
    DisablePlayerInput()
  END IF
END
C# 
public enum RaceState { PreRace, Racing, Finished }
public RaceState raceState = RaceState.PreRace;

public void FinishRace()
{
    if (raceState == RaceState.Racing)
    {
        raceState = RaceState.Finished;
        resultsPanel.SetActive(true);
        player.enabled = false;
    }
}

Call FinishRace() from LapManager once lapTimes.Count reaches your required lap total — the exact list you've been building since Sprint 2.

3 Live Position Tracking Produce

To show whether the player is 1st or 2nd, you need a single number that captures race progress for both karts — not just lap count, since two karts on the same lap could still be far apart on the track. Combine completed laps and current checkpoint into one comparable score.

Pseudocode 
BEGIN GetPlayerPosition()
  playerProgress = playerLaps * totalCheckpoints + playerCheckpointIndex
  opponentProgress = opponentLaps * totalCheckpoints + opponentCheckpointIndex
  IF playerProgress > opponentProgress THEN
    RETURN 1
  ELSE
    RETURN 2
  END IF
END
C# 
public int GetPlayerPosition()
{
    int playerProgress   = playerLaps * totalCheckpoints + playerCheckpointIndex;
    int opponentProgress = opponentLaps * totalCheckpoints + opponentCheckpointIndex;

    return (playerProgress > opponentProgress) ? 1 : 2;
}

Multiplying laps by totalCheckpoints means one full extra lap always outweighs any amount of same-lap checkpoint progress — exactly matching how a real race ranks competitors.

4 Results Screen & UI Polish Produce

Build a Start panel (shown during PreRace), a live position indicator during Racing, and a Results panel shown at Finished — listing each recorded lap time, the total time, and final position, with a "Race Again" button wired to SceneManager.LoadScene.
