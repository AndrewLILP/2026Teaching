Animating Shaders with Time

Everything you've built so far in this series looks the same in every frame. This lesson changes that — a texture that scrolls continuously, and a glow that pulses smoothly — using a single built-in node that tracks elapsed time. No C# script drives any of this; the animation lives entirely inside the material itself.

Producing & Implementing

Build Animated Effects

Predict your node plan in pseudocode first, then compare against the annotated graph recipe below.

1 Scrolling Texture Produce

Multiply the Time node's output by an exposed scrollSpeed Float, then add the result into your UV coordinates before they reach Sample Texture 2D — continuously shifting which part of the texture is being read.

Pseudocode 
BEGIN BuildScroll()
  CREATE Time node
  CREATE Float node (scrollSpeed), EXPOSED = true
  scrollAmount = Time * scrollSpeed
  scrolledUV = UV + scrollAmount
  CONNECT scrolledUV TO SampleTexture2D.UV
END
ShaderGraph Nodes 
Time ──▶ Multiply (× scrollSpeed)
UV   ──▶ Add ◀── Multiply.Out
Add.Out ──▶ Sample Texture 2D.UV
2 Pulsating Emission Produce

Feed Time (multiplied by an exposed pulseSpeed) into a Sine node, remap its -1 to 1 output into 0 to 1, then multiply that against your HDR Color before it reaches Emission — smoothly brightening and dimming the glow.

Pseudocode 
BEGIN BuildPulse()
  CREATE Float node (pulseSpeed), EXPOSED = true
  wave = Sine(Time * pulseSpeed)
  brightness = Remap(wave, -1..1, 0..1)
  pulsedGlow = HDRColor * brightness
  CONNECT pulsedGlow TO Fragment.Emission
END
ShaderGraph Nodes 
Time ──▶ Multiply (× pulseSpeed) ──▶ Sine
Sine.Out ──▶ Remap (-1..1 → 0..1)
Remap.Out ──▶ Multiply ◀── HDR Color
Multiply.Out ──▶ Fragment.Emission
Why Sine, not a straight ramp?

A linear ramp that resets abruptly from maximum back to zero produces a visible "snap" every cycle. Sine rises and falls smoothly and continuously with no sudden jump, because it naturally loops back on itself — exactly the shape a convincing pulse needs.

3 Independent, Exposed Speeds Produce

Confirm scrollSpeed and pulseSpeed are both exposed properties, adjustable independently in the Inspector — the same reusable-property idea from Lessons 1 and 2, now applied twice in one graph.

4 Consider the Cost Produce

Every one of these nodes runs for every pixel, every frame, for every object using this material. That's fine for a handful of glowing objects — it becomes a real performance question if a scene has hundreds of them animating simultaneously. Note this trade-off in your notebook rather than treating it as a hidden cost.

Testing & Evaluating

Test Your Shader

Animated effects need to be watched over time, not judged from a single still frame.

Test CaseStepsExpected ResultActual Result
Scroll loops cleanly Watch the texture scroll continuously for at least 10 seconds Motion is continuous with no visible jump, reset, or stutter Record here

Pulse is smooth (negative test) Watch the glow pulse for at least one full cycle Brightness rises and falls smoothly — no sudden snap back to a starting value Record here

Before this counts as complete, all of the following must be true:
