Unlit Colour & Transparency

Every material you've used so far in Galaxy or Kart — including the collectible orb's glow — was built by someone else in ShaderGraph. This series pulls back the curtain: you'll learn to build materials node by node, starting with the simplest possible shader, an Unlit graph with a solid colour and adjustable transparency.

About the Shader Series

Standalone Optional Extension

The Shader Series sits alongside Galaxy and Kart rather than inside either one — it works with whichever path you chose, since every Unity project uses the same ShaderGraph system underneath. Identifying & Defining

Lesson 1 Brief

Studio context: "Every visual effect in this studio starts as a graph, not a photo or a filter. Before you build anything fancy, prove you understand the simplest possible shader — solid colour in, transparency controlled on purpose. Everything else in this series builds on exactly this."

You'll need Unity's URP (Universal Render Pipeline) and the ShaderGraph package, both already set up in your Galaxy/Kart project. Work in a new, empty Shader Graph asset — don't edit the collectible orb's existing material directly.

Identifying & Defining

Deconstruct an Existing Material

Back in Lessons 1–2, you noted that the collectible orb uses a ShaderGraph material called UnlitGlow. Open it now (double-click the material's Shader Graph asset) and look at the node graph without changing anything.

What to look for

Find the Fragment block on the right-hand side of the graph — this is where nodes eventually connect to decide what colour and transparency each pixel of the surface should be. Trace backward from Base Color and Alpha to see what feeds into them. You don't need to understand every node yet — just confirm that colour and transparency are separate inputs, fed by separate chains of nodes.

Producing & Implementing

Build an Unlit Colour Shader

Create a new Shader Graph asset and set its type to Unlit — meaning it ignores scene lighting entirely, which keeps this first shader simple to reason about. Predict your node plan in pseudocode first, then compare against the annotated graph recipe below.

1 Solid Colour into Base Color Produce

Add a Color node, set it to a colour of your choice, and connect its output directly into the Fragment block's Base Color input.

Pseudocode 
BEGIN BuildBaseColour()
  CREATE Color node (myColour)
  CONNECT myColour TO Fragment.BaseColor
END
ShaderGraph Nodes 
Color node
  └─ Out (RGBA) ──▶ Fragment.Base Color
2 Controllable Alpha Produce

Add a Float node (a single adjustable number) and connect it into the Fragment block's Alpha input. Expose it as a property so it can be adjusted per-material in the Inspector, without editing the graph itself.

Pseudocode 
BEGIN BuildAlpha()
  CREATE Float node (transparency), EXPOSED = true
  CONNECT transparency TO Fragment.Alpha
END
ShaderGraph Nodes 
Float node (exposed: Transparency)
  └─ Out ──▶ Fragment.Alpha

Exposing the Float as a property means every material built from this graph can have its own transparency value — one instance fully solid, another half see-through — without needing a second copy of the graph.

3 Set Surface Type to Transparent Produce

In the Graph Settings panel, change Surface Type from Opaque to Transparent. Save the graph, create a Material from it, and apply it to a test object in your scene.

Why this step is easy to forget

Alpha is connected and exposed after Step 2, but with Surface Type left on Opaque, it will have no visible effect at all — the object will render fully solid no matter what value you set. This is the shader equivalent of a guard condition: the connection exists, but a separate setting has to also allow it to matter.

Testing & Evaluating

Test Your Shader

A shader that only looks right in one specific scene isn't finished — test it somewhere it wasn't originally built for.

Test CaseStepsExpected ResultActual Result
Colour renders correctly Apply the material to a test object with Alpha set to 1 Object displays exactly the chosen Base Color, fully solid Record here
Transparency scales correctly Adjust the exposed transparency value between 0 and 1 in the Inspector Object visibly fades smoothly from invisible to fully solid Record here
Surface Type check (negative test) Temporarily switch Surface Type back to Opaque with a partial Alpha value set Object renders fully solid regardless of the Alpha value, confirming Alpha only matters when Surface Type is Transparent Record here
Works on either path's object Apply the same material to a Galaxy collectible-style object and a Kart-style object The material behaves identically on both — it isn't tied to one path's specific project setup Record here
Lesson Checkpoint

Before moving to the next lesson in the series, all of the following must be true:

  • A working Unlit Shader Graph exists with Base Color and Alpha both connected.
  • Transparency is exposed as a property and adjustable per-material without editing the graph.
  • Surface Type is set to Transparent, and you can explain what would happen if it wasn't.
  • All four test cases above have recorded actual results.
