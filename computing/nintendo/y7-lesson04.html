Identifying & Defining

Data & Variables — Score is Information

A game score isn't just a number on a screen — it is data. Understanding how games store and display data is a core computing skill that applies far beyond game development.

🗄️ Data concept

Data is raw information that a program stores, processes, and outputs. In your game, the score is data — it starts at zero, is updated when events occur, and is displayed to the player.

Variables are named storage locations that hold data values which can change. score, lives, and playerX are all variables. In MakeCode Arcade, info.score() manages the score as a built-in variable.

Good data management means: storing data in the right place, updating it correctly when events happen, and displaying it clearly to the user. All three apply to your score system.

Input → Process → Output

Your score system is a classic IPO model:

Input

Player overlaps a collectible sprite

→
Process

Score variable increases by 1, collectible is destroyed

→
Output

Updated score displays on screen, sound plays

Now go back to your analysis task — you can answer Question 4 now. info.score() in MakeCode Arcade manages the score internally. You don't need to declare a separate variable — but you still control when it changes and by how much.

Producing & Implementing

Sprint 2 — Four Steps

Open your MakeCode Arcade project from Lesson 3. Your Sprint 1 code should still be there. Add these four steps on top of it — don't replace anything from Sprint 1.

1
Create a Collectible Sprite
Sprites

A collectible is a sprite of a different kind from the player. MakeCode Arcade has a built-in kind called Food — we'll use that, or you can create a custom kind that matches your game theme (e.g. Coin, Star, PowerUp).

Block: Inside on start, after your existing Sprint 1 code, add:

🟨 Sprites → set coin to sprite [ ] of kind Food

Draw a simple collectible image — a coin circle, a star, or a token that matches your theme. Keep it clearly different from your player sprite.

Then set its position in the level:

📍 Sprites → set coin position to x [80] y [80]
Your Pseudocode 
BEGIN createCollectible
  SET coin ← new sprite
  SET coin.kind ← Food
  SET coin.x ← 80
  SET coin.y ← 80
END createCollectible
JavaScript 
let coin = sprites.create(
  img`your image`,
  SpriteKind.Food
)
coin.x = 80
coin.y = 80
Placing multiple collectibles Repeat the create + position blocks for each collectible. Use different x/y values to spread them across your level — on platforms, in gaps that require jumping, or at the end of paths. Placement is a design decision: where you put collectibles teaches the player how to explore your world.

Aim for at least 5 collectibles spread across different parts of your tilemap.

2
Detect the Overlap & Update Score
Events

An overlap event fires when two sprites of specified kinds touch. This is the most important event in most platformers — it handles collection, damage, winning, and losing.

Block: Add a new event block (this goes outside on start, as a separate top-level block):

💥 Sprites → on sprite of kind Player overlaps otherSprite of kind Food

Inside that event, add:

📊 Info → change score by 1
Your Pseudocode 
IF player overlaps collectible
THEN
  score ← score + 1
ENDIF
JavaScript 
sprites.onOverlap(
  SpriteKind.Player,
  SpriteKind.Food,
  function(sprite, otherSprite) {
    info.changeScoreBy(1)
  }
)

Test it — walk your player into a collectible. The score counter should appear in the top corner and increment. The collectible stays on screen for now — Step 3 fixes that.

🗄️ Data note

info.changeScoreBy(1) adds 1 to MakeCode Arcade's internal score variable and updates the display automatically. This is an example of the game processing data (updating the stored score) and outputting data (displaying the new value) in one function call.

3
Destroy the Collectible
Sprites

A collectible that stays on screen after being collected breaks the game — the player would earn infinite points by standing still. destroy() removes a sprite from the game permanently.

Inside the same overlap event as Step 2, add directly below change score by 1:

💥 Sprites → destroy otherSprite
Your Pseudocode 
IF player overlaps collectible
THEN
  score ← score + 1
  collectible.destroy()
ENDIF
JavaScript 
sprites.onOverlap(
  SpriteKind.Player,
  SpriteKind.Food,
  function(sprite, otherSprite) {
    info.changeScoreBy(1)
    otherSprite.destroy()
  }
)

Notice the parameter is otherSprite — not coin. This is important: the overlap event receives a reference to whichever specific collectible was touched. Using otherSprite means the function works for any Food sprite, not just the first one you created.

Test it — collecting should now remove the sprite from the screen.

4
Add Feedback — Sound & Effect
Feedback

Collecting something should feel satisfying. Without feedback, players miss collections or aren't sure they happened. At minimum, add a sound. For a stronger effect, add a visual burst too.

Sound Feedback

Inside the same overlap event, add below destroy:

🔊 Music → play sound ba ding until done

Visual Effect (Optional but recommended)

✨ Sprites → spray effects from otherSprite

Add the effect block before destroy — if you destroy first, there's no sprite to play the effect on.

Your Pseudocode 
IF player overlaps collectible
THEN
  score ← score + 1
  OUTPUT visual effect
  collectible.destroy()
  OUTPUT sound
ENDIF
JavaScript 
sprites.onOverlap(
  SpriteKind.Player,
  SpriteKind.Food,
  function(sprite, otherSprite) {
    info.changeScoreBy(1)
    otherSprite.startEffect(
      effects.spray
    )
    otherSprite.destroy()
    music.play(
      music.melodyPlayable(
        music.baDing
      ),
      music.PlaybackMode.UntilDone
    )
  }
)

Final test — collect every item in your level. Score should increment correctly for each one. Now think about your win condition: should the game end when all collectibles are collected? If so, you can add game.over(true) when the score reaches your target total.

✓ Sprint 2 — Checklist

Your game must be able to:

  • Display at least five collectible sprites placed across the level
  • Increment the score by 1 each time the player touches a collectible
  • Remove each collectible from the screen after it is collected
  • Play a sound when a collectible is collected
  • Show the score counter visibly on screen during gameplay
  • Allow all Sprint 1 behaviours to still work (movement, jumping, gravity, platforms)

Debugging Tip

Something Not Working?

  1. Score doesn't change: Is your overlap event using the correct sprite kinds? Both sides of the overlap must match — Player and Food (or whatever kind you chose).
  2. Collectible doesn't disappear: Are you using otherSprite.destroy() not coin.destroy()? If you named a specific sprite, only that one will be destroyed — all others stay.
  3. Effect doesn't show: Move startEffect to before destroy() — you can't add an effect to a sprite that's already been removed.
  4. Sprint 1 broke: Check that all Sprint 2 blocks are separate from your Sprint 1 on start blocks — the overlap event should be its own top-level block, not nested inside on start.
